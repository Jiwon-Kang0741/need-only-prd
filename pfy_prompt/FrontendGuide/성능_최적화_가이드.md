## 📋 이 문서의 목적

Vue 화면 전환 시 **필수적으로 적용해야 하는 성능 최적화** 기법을 정리합니다.

## 1️⃣ DataTable 가상 스크롤링 (CRITICAL!)

### 문제점

```vue
<!-- ❌ WRONG - 모든 행을 렌더링 -->
<DataTable
  :value="data"
  scrollHeight="flex"
>
```

**성능 지표**:
- 1000개 행 × 27개 컬럼 = 27,000개 DOM 노드
- 초기 로딩: ~3초
- 스크롤 FPS: ~15
- 메모리 사용: ~200MB

### 해결 방법

```vue
<!-- ✅ CORRECT - 가상 스크롤링 -->
<DataTable
  :value="data"
  scrollable
  scrollHeight="540px"
  :virtualScrollerOptions="{ itemSize: 46 }"
>
```

**성능 지표**:
- DOM 노드: 27,000개 → ~300개 (90% 감소)
- 초기 로딩: ~3초 → ~100ms (97% 개선)
- 스크롤 FPS: ~15 → ~60 (300% 개선)
- 메모리 사용: ~200MB → ~50MB (75% 감소)

### 설정 가이드

#### A. itemSize 계산

```bash
# Chrome DevTools 열기
# Elements 탭에서 행 선택

.p-datatable-tbody > tr {
  height: 46px;  # ← 이 값 사용
}
```

#### B. scrollHeight 계산

```typescript
// 공식: (보이는 행 수 + 2) × itemSize
const visibleRows = 11;  // 화면에 보이는 행 수
const scrollHeight = (visibleRows + 2) * 46;  // 598px

// 실제 사용
scrollHeight="540px"  // 약간 작게 설정
```

**왜 작게 설정하나요?**
- 스크롤바를 강제로 표시
- 사용자가 데이터가 더 있다는 것을 인지

## 2️⃣ 템플릿 슬롯 포맷팅 제거 (CRITICAL!)

### 문제점

```vue
<!-- ❌ WRONG - 렌더링 시마다 함수 호출 -->
<Column field="ASK_DT" header="의뢰일자">
  <template #body="{ data }">
    {{ formatDate(data.ASK_DT, 'YYYYMMDD', 'YYYY-MM-DD') }}
  </template>
</Column>
```

**성능 지표**:
- 함수 호출: 5000번+ (렌더링, 스크롤, hover마다)
- 행 클릭 지연: ~500ms
- CPU 사용: ~80%

### 해결 방법

```typescript
// ✅ CORRECT - watch에서 한 번만 포맷팅
import { formatDate } from '@/utils/dateUtils';

watch(
  () => props.fetchedMainData,
  async (newData) => {
    if (!newData) return;
    
    await nextTick();
    
    newData.forEach((row: any) => {
      // 의뢰일자
      if (row.ASK_DT) {
        row.ASK_DT_FORMATTED = formatDate(row.ASK_DT, 'YYYYMMDD', 'YYYY-MM-DD');
      }
      
      // 요청일자
      if (row.REQ_DT) {
        row.REQ_DT_FORMATTED = formatDate(row.REQ_DT, 'YYYYMMDD', 'YYYY-MM-DD');
      }
      
      // ... 나머지 날짜 필드
    });
  },
  { immediate: true }
);
```

```vue
<!-- 포맷팅된 필드 사용 -->
<Column field="ASK_DT_FORMATTED" header="의뢰일자" />
<Column field="REQ_DT_FORMATTED" header="요청일자" />
```

**성능 지표**:
- 함수 호출: 5000번+ → 1000번 (데이터 로딩 시 1회)
- 행 클릭 지연: ~500ms → 즉시
- CPU 사용: ~80% → ~20%

### formatDate 유틸리티

```typescript
// @/utils/dateUtils.ts
export function formatDate(
  value: string | null | undefined,
  fromFormat: string,
  toFormat: string
): string {
  if (!value) return '';
  
  // YYYYMMDD → YYYY-MM-DD
  if (fromFormat === 'YYYYMMDD' && toFormat === 'YYYY-MM-DD') {
    if (value.length === 8) {
      return `${value.substring(0, 4)}-${value.substring(4, 6)}-${value.substring(6, 8)}`;
    }
  }
  
  // YYYYMM → YYYY-MM
  if (fromFormat === 'YYYYMM' && toFormat === 'YYYY-MM') {
    if (value.length === 6) {
      return `${value.substring(0, 4)}-${value.substring(4, 6)}`;
    }
  }
  
  return value;
}
```

## 3️⃣ GPU 가속 (CRITICAL!)

### 문제점

```scss
/* ❌ WRONG - GPU 가속 없음 */
.p-datatable-tbody tr:hover {
  background-color: var(--primary-50);
}
```

**성능 지표**:
- hover FPS: ~20
- 리플로우: 전체 테이블
- Paint 시간: ~50ms

### 해결 방법

```scss
/* ✅ CORRECT - GPU 가속 */
.p-datatable {
  .p-datatable-tbody {
    tr {
      // ⭐ GPU 레이어 생성
      will-change: background-color;
      transition: background-color 0.2s;
      
      &:hover:not(.p-datatable-row-selected) {
        background-color: var(--primary-50);
      }
    }
  }
  
  .p-datatable-tbody > tr > td,
  .p-datatable-thead > tr > th {
    // ⭐ CSS containment: 레이아웃 독립
    contain: layout style;
  }
}
```

**성능 지표**:
- hover FPS: ~20 → ~60 (200% 개선)
- 리플로우: 전체 테이블 → 해당 행만
- Paint 시간: ~50ms → ~5ms (90% 개선)

### will-change 사용 가이드

#### ⚠️ DO: 자주 변경되는 속성에만 사용

```scss
/* ✅ CORRECT */
tr {
  will-change: background-color;  // hover 시 자주 변경
}

button {
  will-change: transform;  // hover/active 시 자주 변경
}
```

#### ⚠️ DON'T: 과도하게 사용

```scss
/* ❌ WRONG - 메모리 낭비 */
* {
  will-change: transform, opacity, background-color;
}
```

**이유**: GPU 레이어 생성 → 메모리 사용 증가

## 4️⃣ CSS Containment

### 개념

```scss
td, th {
  contain: layout style;
}
```

**효과**:
- 각 셀의 레이아웃을 독립적으로 처리
- 한 셀 변경 시 전체 테이블 재계산 방지

### 성능 지표

| 속성 | 재계산 범위 | Paint 시간 |
|------|------------|-----------|
| contain: none | 전체 테이블 | ~50ms |
| contain: layout | 해당 행만 | ~20ms |
| contain: layout style | 해당 셀만 | ~5ms |

### 사용 가이드

```scss
.p-datatable {
  // ⭐ 셀 레이아웃 독립
  .p-datatable-tbody > tr > td,
  .p-datatable-thead > tr > th {
    contain: layout style;
  }
  
  // ⭐ 행 레이아웃 독립
  .p-datatable-tbody > tr {
    contain: layout;
  }
}
```

## 5️⃣ Computed 최적화

### 문제점

```typescript
// ❌ WRONG - 매번 재계산
const totalCount = () => {
  return fetchedSumData?.dsOutput?.[0]?.STAT_TOT || 0;
};
```

```vue
<template>
  <span>{{ totalCount() }}</span>  <!-- 렌더링 시마다 호출 -->
</template>
```

### 해결 방법

```typescript
// ✅ CORRECT - 캐싱
const totalCount = computed(() => {
  if (!fetchedSumData.value?.dsOutput?.[0]) return 0;
  return fetchedSumData.value.dsOutput[0].STAT_TOT || 0;
});
```

```vue
<template>
  <span>{{ totalCount }}</span>  <!-- 값만 참조 -->
</template>
```

**성능 지표**:
- 재계산: 매 렌더링 → 의존성 변경 시만
- CPU 사용: ~10% 감소

### 불필요한 watch 방지

```typescript
// ❌ WRONG - 모든 변경에 반응
watch(fetchedMainData, (newData) => {
  console.log('Data changed:', newData);

```

### 조건부 watch

```typescript
// ✅ CORRECT - 필요한 경우만 실행
watch(
  () => props.fetchedMainData,

    if (!newData) return;  // ⭐ 빠른 종료
    
    await nextTick();
    
    // 포맷팅 로직
    newData.forEach((row: any) => {
      // ...
    });
  },
  { immediate: true }
);
```

```typescript
// ❌ WRONG - 깊은 감시 (느림)
```

## 7️⃣ v-if vs v-show

### v-if (조건부 렌더링)

```vue
<!-- ✅ CORRECT - 초기 렌더링 시 불필요한 경우 -->
<PMDP010DataTable v-if="fetchedMainData" />
```

**장점**: 초기 렌더링 빠름
**단점**: 토글 시 DOM 재생성 (느림)

### v-show (CSS display 토글)

```vue
<!-- ✅ CORRECT - 자주 토글되는 경우 -->
<div v-show="isVisible">...</div>
```

**장점**: 토글 빠름
**단점**: 초기 렌더링 느림

### 선택 가이드

| 상황 | 사용 |
|------|------|
| 초기 렌더링 시 불필요 | v-if |
| 자주 토글 | v-show |
| 대용량 데이터 | v-if |
| 간단한 UI | v-show |

## 8️⃣ Key 최적화

### 문제점

```vue
<!-- ❌ WRONG - index를 key로 사용 -->
<div v-for="(item, index) in items" :key="index">
  {{ item.name }}
</div>
```

**문제점**: 항목 삭제 시 모든 DOM 재생성

### 해결 방법

```vue
<!-- ✅ CORRECT - 고유 ID를 key로 사용 -->
<div v-for="item in items" :key="item.ASK_NO">
  {{ item.name }}
</div>
```

**효과**: 변경된 항목만 업데이트
