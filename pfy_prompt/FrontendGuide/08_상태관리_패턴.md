# 08. 상태관리 패턴

## 📋 이 단계의 목적

Vue 3 Composition API를 활용한 **효율적인 상태관리 패턴**을 구현합니다.

---

## 1️⃣ searchParams 패턴 (CRITICAL!)

### 1.1 개념

**searchParams**는 화면의 모든 검색 조건을 담는 중앙 상태입니다.

```typescript
// index.vue
const searchParams = ref<PMDP010SearchParams>({
  prg_stat: null,
  canc_stat: null,
  cont_no: '',
  searchDt: [],
  // ...
});

provide('searchParams', searchParams);
```

### 1.2 데이터 흐름

```
┌────────────────────────────────────────┐
│           index.vue (Parent)           │
│                                        │
│  searchParams = ref({})                │
│  provide('searchParams', searchParams) │
└────────┬────────────────┬──────────────┘
         │                │
         │ inject         │ inject
         ▼                ▼
┌──────────────┐   ┌──────────────┐
│ SearchForm   │   │   SumGrid    │
│              │   │              │
│ watch(       │◄──│ handleClick  │
│  prg_stat    │   │   {          │
│ )            │   │   searchParams
│              │   │   .prg_stat  │
└──────────────┘   │   = 'S01'    │
                   │   }          │
                   └──────────────┘
```

### 1.3 왜 searchParams인가?

#### ✅ 장점

1. **단일 진실의 원천** (Single Source of Truth)
   - 모든 컴포넌트가 같은 상태를 참조
   - 동기화 문제 없음

2. **간단한 업데이트**
   ```typescript
   // ✅ 간단
   searchParams.value.prg_stat = 'S01';
   ```

3. **자동 전파**
   - inject한 모든 컴포넌트에 자동 반영
   - watch가 변경 감지

4. **API 호출 용이**
   ```typescript
   const handleSearch = async () => {
     await fetchData(searchParams.value);  // 그대로 전달
   };
   ```

#### ❌ 대안의 문제점

```typescript
// ❌ WRONG - 여러 개의 ref
const prgStat = ref('');
const cancStat = ref('');
const contNo = ref('');
// → 관리 복잡, 동기화 어려움

// ❌ WRONG - 각 컴포넌트가 자체 상태
// SearchForm: form = { prg_stat: 'S01' }
// SumGrid: selectedStat = 'S01'
// → 동기화 복잡, 버그 발생 가능
```

## 2️⃣ provide/inject 패턴

### 2.1 기본 사용법

#### Parent (index.vue)

```vue
<script lang="ts" setup>
import { ref, provide } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

const searchParams = ref<PMDP010SearchParams>({});

// ⭐ provide로 자식에게 전달
provide('searchParams', searchParams);
</script>
```

#### Child (SearchForm, SumGrid 등)

```vue
<script lang="ts" setup>
import { inject } from 'vue';
import type { Ref } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

// ⭐ inject로 부모에서 받기
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;

// 사용
const handleClick = () => {
  searchParams.value.prg_stat = 'S01';
};
</script>
```

### 2.2 타입 안전성

```typescript
// ⚠️ DO: 타입 지정
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;

// ❌ DON'T: any 사용
const searchParams = inject('searchParams') as any;
```

### 2.3 여러 상태 전달

```vue
<script lang="ts" setup>
// Parent
const searchParams = ref({});
const loading = ref(false);
const selectedRows = ref([]);

provide('searchParams', searchParams);
provide('loading', loading);
provide('selectedRows', selectedRows);
</script>
```

```vue
<script lang="ts" setup>
// Child
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;
const loading = inject<Ref<boolean>>('loading')!;
const selectedRows = inject<Ref<any[]>>('selectedRows')!;
</script>
```

## 3️⃣ watch 패턴 (CRITICAL!)

### 3.1 SearchForm 동기화 패턴

```typescript
// PMDP010SearchForm.vue
import { watch } from 'vue';

// ⭐⭐⭐ CRITICAL: SelectBox 동기화
watch(
  () => searchParams.value?.prg_stat,
  (newVal) => {
    // ⚠️ 조건: newVal !== undefined
    if (newVal !== undefined && searchFormRef.value?.form) {
      // ⚠️ Ref 접근: .value.form.setFieldValue
      searchFormRef.value.form.setFieldValue('prg_stat', newVal ?? '');
    }
  }
);
```

### 3.2 watch 옵션

```typescript
// immediate: 초기 실행
watch(
  () => searchParams.value?.prg_stat,
  (newVal) => { /* ... */ },
  { immediate: true }  // ⭐ 컴포넌트 마운트 시 즉시 실행
);

// deep: 깊은 감시
watch(
  searchParams,
  (newVal) => { /* ... */ },
  { deep: true }  // ⚠️ 성능 저하 가능, 가급적 사용 안 함
);

// flush: 실행 시점
watch(
  () => props.fetchedMainData,
  async (newData) => {
    await nextTick();  // DOM 업데이트 후 실행
    // 포맷팅 로직
  },
  { flush: 'post' }  // 'pre' (기본), 'post', 'sync'
);
```

### 3.3 여러 값 감시

```typescript
// ⭐ 배열로 여러 값 감시
watch(
  [
    () => searchParams.value?.prg_stat,
    () => searchParams.value?.canc_stat,
  ],
  ([newPrgStat, newCancStat]) => {
    console.log('prg_stat:', newPrgStat);
    console.log('canc_stat:', newCancStat);
  }
);
```

### 3.4 watchEffect (자동 의존성)

```typescript
import { watchEffect } from 'vue';

// ⭐ 사용된 모든 ref를 자동으로 감시
watchEffect(() => {
  console.log('prg_stat:', searchParams.value.prg_stat);
  console.log('canc_stat:', searchParams.value.canc_stat);
  // → 둘 중 하나만 변경되어도 실행
});
```

## 4️⃣ Computed 패턴

### 4.1 기본 사용

```typescript
import { computed } from 'vue';

// ⭐ 캐싱된 계산 속성
const totalCount = computed(() => {
  if (!fetchedSumData.value?.dsOutput?.[0]) return 0;
  return fetchedSumData.value.dsOutput[0].STAT_TOT || 0;
});
```

### 4.2 복잡한 계산

```typescript
const progressItems = computed(() => {
  if (!fetchedSumData.value?.dsOutput?.[0]) {
    return [
      { field: 'S01', label: '저장', count: 0 },
      { field: 'S02', label: '의뢰', count: 0 },
      { field: 'S03', label: '요청확정', count: 0 },
      { field: 'S04', label: '요청철회', count: 0 },
    ];
  }
  
  const data = fetchedSumData.value.dsOutput[0];
  
  return [
    { field: 'S01', label: '저장', count: data.STAT_SAVE || 0 },
    { field: 'S02', label: '의뢰', count: data.STAT_ASK || 0 },
    { field: 'S03', label: '요청확정', count: data.STAT_REQ || 0 },
    { field: 'S04', label: '요청철회', count: data.STAT_CANC || 0 },
  ];
});
```

### 4.3 Computed vs Function

```typescript
// ❌ WRONG - 함수 (매번 실행)
const getTotalCount = () => {
  return fetchedSumData.value?.dsOutput?.[0]?.STAT_TOT || 0;
};

// ✅ CORRECT - computed (캐싱)
const totalCount = computed(() => {
  return fetchedSumData.value?.dsOutput?.[0]?.STAT_TOT || 0;
});
```

**차이점**:
- 함수: 참조할 때마다 실행
- computed: 의존성 변경 시에만 재계산

## 7️⃣ 로딩 상태 관리

### 7.1 기본 패턴

```typescript
const loading = ref(false);

const fetchData = async () => {
  try {
    loading.value = true;
    const response = await api.post<ApiResponse<any>>('/api/v1/PMDP010-selectDpAskList', params);
```

## 8️⃣ 에러 상태 관리

### 8.1 에러 핸들링 패턴

```typescript
const error = ref<string | null>(null);

const fetchData = async () => {
  try {
    loading.value = true;
    error.value = null;  // ⭐ 에러 초기화
    
    const response = await api.post<ApiResponse<any>>('/api/v1/PMDP010-selectDpAskList', params);
    if (response.data.header.responseCode === 'S0000') {
      fetchedMainData.value = response.data.payload;
    }
  } catch (err: any) {
    // ⭐ 에러 메시지 설정
    error.value = err.response?.data?.message || '데이터 조회 중 오류가 발생했습니다.';
    console.error('데이터 조회 실패:', err);
  } finally {
    loading.value = false;
  }
};
```

## 🔟 상태관리 안티패턴

### ⚠️ DON'T: 중복 상태

```typescript
// ❌ WRONG - 같은 데이터를 여러 곳에 저장
// index.vue
const searchParams = ref({ prg_stat: 'S01' });

// SearchForm.vue
const formData = ref({ prg_stat: 'S01' });  // ← 중복!

// SumGrid.vue
const selectedStat = ref('S01');  // ← 중복!
```

**문제점**: 동기화 복잡, 버그 발생 가능

### ⚠️ DON'T: Props drilling

```vue
<!-- ❌ WRONG - Props를 여러 단계로 전달 -->
<Parent :searchParams="searchParams">
  <Child1 :searchParams="searchParams">
    <Child2 :searchParams="searchParams">
      <Child3 :searchParams="searchParams">
        <!-- 사용 -->
      </Child3>
    </Child2>
  </Child1>
</Parent>
```

**해결**: provide/inject 사용

### ⚠️ DON'T: 직접 수정

```typescript
// ❌ WRONG - inject한 상태를 직접 교체
const searchParams = inject<Ref<any>>('searchParams')!;
searchParams.value = { ... };  // ← 전체 교체 금지!

// ✅ CORRECT - 속성만 변경
searchParams.value.prg_stat = 'S01';
```
