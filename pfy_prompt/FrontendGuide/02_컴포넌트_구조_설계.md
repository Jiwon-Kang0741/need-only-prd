## 📝 Step 2: SearchForm 설계

### 2.1 컴포넌트 구조

```vue
<template>
  <SearchForm ref="searchFormRef" v-bind="searchFormConfig">
    <template #additionalButtons>
      <Button label="조회" @click="handleSearch" />
    </template>
  </SearchForm>
</template>

<script lang="ts" setup>
import { ref, inject, watch } from 'vue';
import type { Ref } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

// ⭐ inject searchParams
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;

// ⭐ SearchForm ref 접근
const searchFormRef = ref();

// ⭐⭐⭐ CRITICAL: watch로 SelectBox 동기화
watch(
  () => searchParams.value?.prg_stat,
  (newVal) => {
    // ⚠️ DO: newVal !== undefined 조건 사용
    if (newVal !== undefined && searchFormRef.value?.form) {
      // ⚠️ DO: .value.form.setFieldValue (NOT .value.value.form)
      searchFormRef.value.form.setFieldValue('prg_stat', newVal ?? '');
    }
  }
);
</script>
```

### 2.2 핵심 패턴 설명

#### A. watch 조건문

```typescript
// ❌ WRONG - null과 빈 문자열을 놓침
if (newVal) { ... }

// ❌ WRONG - null도 업데이트해야 함
if (newVal !== null) { ... }

// ✅ CORRECT - undefined만 제외
if (newVal !== undefined) { ... }
```

**이유**:
- `prg_stat = null`: 전체 보기 (조건 제거)
- `prg_stat = ''`: 빈 값 필터
- `prg_stat = undefined`: 초기 상태 (업데이트 불필요)

#### B. Ref 접근 경로

```typescript
// ❌ WRONG - .value.value는 존재하지 않음
searchFormRef.value.value.form.setFieldValue(...)

// ❌ WRONG - .form.value도 틀림
searchFormRef.value.form.value.setFieldValue(...)

// ✅ CORRECT - Vue가 자동으로 unwrap
searchFormRef.value.form.setFieldValue(...)
```

**이유**: `defineExpose({ searchFormRef })`에서 이미 ref를 노출했으므로 `.value`가 한 번만 필요.

```vue
<template>
```

### 4.2 utils/index.ts — TableColumn 타입 사용 (CRITICAL!)

```typescript
// utils/index.ts
import type { TableColumn } from '@/components/common/dataTable2/types';
import type { XxxResDto } from '@/api/pages/{module}/{category}/types';

export type XxxDisplayRow = XxxResDto & {
  someDateFormatted?: string;
};

// ⭐ TableColumn 타입 필수 — 커스텀 타입 정의 금지
```
