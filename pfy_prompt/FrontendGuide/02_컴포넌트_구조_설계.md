## 📝 Step 2: SearchForm 설계

### 2.1 컴포넌트 구조

```vue
<template>
  <SearchForm ref="searchFormRef" v-bind="searchFormConfig">
    <template #additionalButtons>
      <Button label="조회" @click="handleSearch" />
    </template>
  </SearchForm>
</template>

<script lang="ts" setup>
import { ref, inject, watch } from 'vue';
import type { Ref } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

// ⭐ inject searchParams
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;

// ⭐ SearchForm ref 접근
const searchFormRef = ref();

// ⭐⭐⭐ CRITICAL: watch로 SelectBox 동기화
watch(
  () => searchParams.value?.prg_stat,
  (newVal) => {
    // ⚠️ DO: newVal !== undefined 조건 사용
    if (newVal !== undefined && searchFormRef.value?.form) {
      // ⚠️ DO: .value.form.setFieldValue (NOT .value.value.form)
      searchFormRef.value.form.setFieldValue('prg_stat', newVal ?? '');
    }
  }
);
</script>
```

### 2.2 핵심 패턴 설명

#### A. watch 조건문

```typescript
// ❌ WRONG - null과 빈 문자열을 놓침
if (newVal) { ... }

// ❌ WRONG - null도 업데이트해야 함
if (newVal !== null) { ... }

// ✅ CORRECT - undefined만 제외
if (newVal !== undefined) { ... }
```

**이유**:
- `prg_stat = null`: 전체 보기 (조건 제거)
- `prg_stat = ''`: 빈 값 필터
- `prg_stat = undefined`: 초기 상태 (업데이트 불필요)

#### B. Ref 접근 경로

```typescript
// ❌ WRONG - .value.value는 존재하지 않음
searchFormRef.value.value.form.setFieldValue(...)

// ❌ WRONG - .form.value도 틀림
searchFormRef.value.form.value.setFieldValue(...)

// ✅ CORRECT - Vue가 자동으로 unwrap
searchFormRef.value.form.setFieldValue(...)
```

**이유**: `defineExpose({ searchFormRef })`에서 이미 ref를 노출했으므로 `.value`가 한 번만 필요.

## 📝 Step 3: SumGrid (ProgressBar) 설계

### 3.1 컴포넌트 구조

```vue
<template>
  <div class="sum-grid-container">
    <!-- 전체 건수 버튼 -->
    <Button @click="handleTotalClick">

    <Button

      <span>{{ item.label }}</span>

<script lang="ts" setup>
import { inject } from 'vue';
import type { Ref } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

// ⭐⭐⭐ CRITICAL: tomms-lite-front 패턴 사용

  // ⚠️ DO: null 할당 (빈 문자열 아님!)
  searchParams.value.prg_stat = null;
};

const handleProgressClick = (field: string) => {
  // ⚠️ DO: searchParams 직접 업데이트
  searchParams.value.prg_stat = field;
};
</script>
```

### 3.2 핵심 패턴 설명

#### ⚠️ DO: tomms-lite-front 패턴

```typescript
// ✅ CORRECT - 간단하고 명확
const handleProgressClick = (field: string) => {
  searchParams.value.prg_stat = field;
};
```

#### ⚠️ DON'T: 복잡한 ref 체인

```typescript
// ❌ WRONG - 과거 방식, 동작하지 않음
const handleProgressClick = (field: string) => {
  const searchFormRef = inject('searchFormRef');
  searchFormRef.value?.form.setFieldValue('prg_stat', field);
  searchParams.value.prg_stat = field;
};
```

**이유**: `watch`가 알아서 `SelectBox`를 동기화하므로 직접 접근 불필요.

#### ⚠️ DO: null vs 빈 문자열

```typescript
// ✅ CORRECT - 전체 보기
searchParams.value.prg_stat = null;  // WHERE절에서 조건 제외

// ❌ WRONG - 빈 값 필터
searchParams.value.prg_stat = '';    // WHERE A.PRG_STAT = ''
```

## 📝 Step 4: DataTable 설계

### 4.1 컴포넌트 구조

```vue
<template>
  <DataTable
    v-model:selection="selectedRows"
    :value="fetchedMainData"
    scrollable
    scrollHeight="540px"
    :virtualScrollerOptions="{ itemSize: 46 }"
    selectionMode="multiple"
    dataKey="ASK_NO"
  >
    <Column selectionMode="multiple" frozen />
    <Column field="CONT_NO" header="계약번호" />
    <!-- ... -->
  </DataTable>
</template>

<script lang="ts" setup>
import { ref, watch, nextTick } from 'vue';

// ⭐ 가상 스크롤링 필수
const virtualScrollerOptions = { itemSize: 46 };

// ⭐ 고정 높이 필수
const scrollHeight = "540px";

// ⭐ 날짜 포맷팅은 watch에서 처리 (template 슬롯 대신)
watch(fetchedMainData, async (newData) => {
  if (!newData) return;
  
  await nextTick();
  
  newData.forEach((row: any) => {
    if (row.ASK_DT) {
      row.ASK_DT_FORMATTED = formatDate(row.ASK_DT, 'YYYYMMDD', 'YYYY-MM-DD');
    }
  });
}, { immediate: true });
</script>
```

### 4.2 핵심 패턴 설명

#### ⚠️ DO: 가상 스크롤링

```vue
<!-- ✅ CORRECT - 대용량 데이터 성능 최적화 -->
<DataTable
  scrollHeight="540px"
  :virtualScrollerOptions="{ itemSize: 46 }"
>
```

#### ⚠️ DON'T: scrollHeight="flex"

```vue
<!-- ❌ WRONG - 성능 저하 -->
<DataTable scrollHeight="flex">
```

**이유**: `flex`는 모든 행을 렌더링하므로 느림. 고정 높이 + 가상 스크롤링 필수.

#### ⚠️ DO: 날짜 포맷팅 전처리

```typescript
// ✅ CORRECT - watch에서 한 번만 포맷팅
watch(fetchedMainData, async (newData) => {
  await nextTick();
  newData.forEach(row => {
    row.ASK_DT_FORMATTED = formatDate(row.ASK_DT, ...);
  });
});
```

```vue
<Column field="ASK_DT_FORMATTED" header="의뢰일자" />
```

#### ⚠️ DON'T: 템플릿 슬롯 포맷팅

```vue
<!-- ❌ WRONG - 렌더링 시마다 함수 호출 (매우 느림) -->
<Column field="ASK_DT" header="의뢰일자">
  <template #body="{ data }">
    {{ formatDate(data.ASK_DT, 'YYYYMMDD', 'YYYY-MM-DD') }}
  </template>
</Column>
```

**이유**: 1000개 행 × 5개 날짜 컬럼 = 5000번 함수 호출/렌더링.
