# 02. 컴포넌트 구조 설계

## 📋 이 단계의 목적

01단계에서 분석한 내용을 바탕으로 **Vue 컴포넌트 구조**를 설계합니다.

**중요**: 반드시 **다중 파일 구조**를 따라야 합니다! 단일 Vue 파일 금지!

## 🚫 단일 Vue 파일 금지 (CRITICAL!)

```
❌ 금지: 하나의 vue_page에 SearchForm, DataTable, SumGrid, API, 스타일을 모두 포함

```

## 📝 Step 1: index.vue 설계

**index.vue는 오케스트레이터 역할만 합니다!** SearchForm, DataTable, SumGrid, API 호출은 모두 별도 파일로 분리합니다.

### 1.1 기본 구조 (다중 파일 패턴)

```vue
<template>
  <div class="main-content-container">
    <!-- ⭐ ContentHeader는 필수! -->
    <ContentHeader menuId="CpmsEduPondgLst" />
    
    <CpmsEduPondgLstSearchForm @search="handleSearch" />
    <div class="summary-section">
      <CpmsEduPondgLstSumGrid :fetchedSumData="fetchedSumData" />
    </div>
    <div class="data-table-section">
      <CpmsEduPondgLstDataTable :fetchedMainData="fetchedMainData" :loading="loading" />
    </div>
  </div>
</template>

<script lang="ts" setup>
import { provide, ref, onMounted } from 'vue';
import { fetchSelectSummary, fetchSelectList } from '@/api/pages/edu/pondg/cpmsEduPondgLst';
import ContentHeader from '@/components/common/contentHeader/ContentHeader.vue';
import CpmsEduPondgLstSearchForm from './components/cpmsEduPondgLstSearchForm/CpmsEduPondgLstSearchForm.vue';
import CpmsEduPondgLstSumGrid from './components/cpmsEduPondgLstSumGrid/CpmsEduPondgLstSumGrid.vue';
import CpmsEduPondgLstDataTable from './components/cpmsEduPondgLstDataTable/CpmsEduPondgLstDataTable.vue';

const searchParams = ref();
provide('searchParams', searchParams);

const fetchedSumData = ref(null);
const fetchedMainData = ref(null);
const loading = ref(false);

const handleSearch = async () => {
  loading.value = true;
  try {
    const [sum, list] = await Promise.all([
      fetchSelectSummary(searchParams.value),
      fetchSelectList(searchParams.value),
    ]);
    fetchedSumData.value = sum;
    fetchedMainData.value = list;
  } finally {
    loading.value = false;
  }
};

onMounted(() => handleSearch());
</script>

<style scoped lang="scss" src="./cpmsEduPondgLst.scss"></style>
```

**⚠️ 주의**: import 경로의 폴더명도 반드시 camelCase! (예: `cpmsEduPondgLstSearchForm/`)

## 📝 Step 2: SearchForm 설계

### 2.1 컴포넌트 구조

```vue
<template>
  <SearchForm ref="searchFormRef" v-bind="searchFormConfig">
    <template #additionalButtons>
      <Button label="조회" @click="handleSearch" />
    </template>
  </SearchForm>
</template>

<script lang="ts" setup>
import { ref, inject, watch } from 'vue';
import type { Ref } from 'vue';
import type { PMDP010SearchParams } from '@/api/pages/sy/ds/types';

// ⭐ inject searchParams
const searchParams = inject<Ref<PMDP010SearchParams>>('searchParams')!;

// ⭐ SearchForm ref 접근
const searchFormRef = ref();

// ⭐⭐⭐ CRITICAL: watch로 SelectBox 동기화
watch(
  () => searchParams.value?.prg_stat,
  (newVal) => {
    // ⚠️ DO: newVal !== undefined 조건 사용
    if (newVal !== undefined && searchFormRef.value?.form) {
      // ⚠️ DO: .value.form.setFieldValue (NOT .value.value.form)
      searchFormRef.value.form.setFieldValue('prg_stat', newVal ?? '');
    }
  }
);
</script>
```

### 2.2 핵심 패턴 설명

#### A. watch 조건문

```typescript
// ❌ WRONG - null과 빈 문자열을 놓침
if (newVal) { ... }

// ❌ WRONG - null도 업데이트해야 함
if (newVal !== null) { ... }

// ✅ CORRECT - undefined만 제외
if (newVal !== undefined) { ... }
```

**이유**:
- `prg_stat = null`: 전체 보기 (조건 제거)
- `prg_stat = ''`: 빈 값 필터
- `prg_stat = undefined`: 초기 상태 (업데이트 불필요)

#### B. Ref 접근 경로

```typescript
// ❌ WRONG - .value.value는 존재하지 않음
searchFormRef.value.value.form.setFieldValue(...)

// ❌ WRONG - .form.value도 틀림
searchFormRef.value.form.value.setFieldValue(...)

// ✅ CORRECT - Vue가 자동으로 unwrap
searchFormRef.value.form.setFieldValue(...)
```

**이유**: `defineExpose({ searchFormRef })`에서 이미 ref를 노출했으므로 `.value`가 한 번만 필요.

## 📝 Step 4: DataTable 설계

### 4.1 컴포넌트 구조

```vue
<template>
  <DataTable
    v-model:selection="selectedRows"
    :value="fetchedMainData"
    scrollable
    scrollHeight="540px"
    :virtualScrollerOptions="{ itemSize: 46 }"
    selectionMode="multiple"
    dataKey="ASK_NO"
  >
    <Column selectionMode="multiple" frozen />
    <Column field="CONT_NO" header="계약번호" />
    <!-- ... -->
  </DataTable>
</template>

<script lang="ts" setup>
import { ref, watch, nextTick } from 'vue';

// ⭐ 가상 스크롤링 필수
const virtualScrollerOptions = { itemSize: 46 };

// ⭐ 고정 높이 필수
const scrollHeight = "540px";

// ⭐ 날짜 포맷팅은 watch에서 처리 (template 슬롯 대신)
watch(fetchedMainData, async (newData) => {
  if (!newData) return;
  
  await nextTick();
  
  newData.forEach((row: any) => {
    if (row.ASK_DT) {
      row.ASK_DT_FORMATTED = formatDate(row.ASK_DT, 'YYYYMMDD', 'YYYY-MM-DD');
    }
  });
}, { immediate: true });
</script>
```

### 4.2 핵심 패턴 설명

#### ⚠️ DO: 가상 스크롤링

```vue
<!-- ✅ CORRECT - 대용량 데이터 성능 최적화 -->
<DataTable
  scrollHeight="540px"
  :virtualScrollerOptions="{ itemSize: 46 }"

```

#### ⚠️ DON'T: scrollHeight="flex"

```vue
<!-- ❌ WRONG - 성능 저하 -->
<DataTable scrollHeight="flex">
```

**이유**: `flex`는 모든 행을 렌더링하므로 느림. 고정 높이 + 가상 스크롤링 필수.

#### ⚠️ DO: 날짜 포맷팅 전처리

```typescript
// ✅ CORRECT - watch에서 한 번만 포맷팅
watch(fetchedMainData, async (newData) => {
  await nextTick();

    row.ASK_DT_FORMATTED = formatDate(row.ASK_DT, ...);

```

```vue

```

#### ⚠️ DON'T: 템플릿 슬롯 포맷팅

```vue
<!-- ❌ WRONG - 렌더링 시마다 함수 호출 (매우 느림) -->
```
