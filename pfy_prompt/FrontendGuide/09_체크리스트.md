## 📋 이 문서의 목적

**신규 화면 개발** 및 **기존 화면 수정** 완료 전 **최종 점검**을 수행합니다.

---

## ⛔ 0️⃣ 최우선 원칙: 확실하지 않으면 생성 금지!

> **모르는 것을 억지로 만들지 마라. 차라리 만들지 마라.**

- [ ] 컴포넌트명, import 경로, prop 이름이 **가이드/컨텍스트에 명시되어 있는지** 확인 — 없으면 생성 금지
- [ ] API 엔드포인트, HTTP 메서드, 응답 구조가 **가이드에 명시되어 있는지** 확인 — 없으면 생성 금지
- [ ] 백엔드 DTO 필드명, 패키지 경로, 어노테이션이 **제공된 컨텍스트에 있는지** 확인 — 없으면 생성 금지
- [ ] 확실하지 않은 부분은 `// TODO: verify` 주석으로 남기고 코드 생성을 건너뛸 것
- [ ] **불완전하지만 올바른 코드 > 완전하지만 틀린 코드** — 이 원칙을 반드시 준수

## 2️⃣ 파일 네이밍 및 구조 (CRITICAL!)

### 네이밍 규칙
- [ ] screenId가 **camelCase**인지 확인 (모두 소문자 금지!)
  - ❌ `cpmsedupondgedit` → 잘못됨
  - ✅ `cpmsEduPondgEdit` → 올바름
- [ ] 화면 폴더명이 camelCase (예: `cpmsEduPondgEdit/`)
- [ ] 컴포넌트 폴더명이 camelCase + ComponentType (예: `cpmsEduPondgEditSearchForm/`)
- [ ] Vue 컴포넌트 파일이 PascalCase (예: `CpmsEduPondgEditSearchForm.vue`)
- [ ] SCSS 파일이 PascalCase (예: `CpmsEduPondgEditSearchForm.scss`)
- [ ] API 파일이 camelCase (예: `cpmsEduPondgEdit.ts`)

### 다중 파일 구조 (CRITICAL!)
- [ ] **단일 Vue 파일이 아닌** 다중 파일 구조로 생성
- [ ] index.vue는 오케스트레이터 역할만 (SearchForm/DataTable 등을 import)
- [ ] SearchForm, DataTable은 `components/` 하위에 별도 폴더로 분리
- [ ] SumGrid는 **스펙에 진행상태별 집계 필터가 명시된 경우에만** `components/` 하위에 생성 — 단순 목록 화면에는 불필요
- [ ] API 함수는 `api/pages/[module]/[category]/[screenId].ts`에 분리
- [ ] 타입은 `api/pages/[module]/[category]/[screenId]Types.ts`에 정의

### index.vue
- [ ] searchParams ref 생성
- [ ] provide('searchParams', searchParams)
- [ ] 하위 컴포넌트 import 경로 확인 (camelCase 폴더명)
- [ ] 세션 파라미터는 서버측 자동 처리 (프론트에서 추가 불필요)
- [ ] 초기 조회 (onMounted)
- [ ] 에러 처리 (try-catch)

### 폴더 구조 (신규 시)
- [ ] components/[screenId]SearchForm 폴더 생성 (camelCase!)
- [ ] components/[screenId]SumGrid 폴더 생성 (⚠️ 스펙에 진행상태별 집계 필터가 있을 때만! 없으면 생성 금지)
- [ ] components/[screenId]DataTable 폴더 생성 (camelCase!)
- [ ] 각 폴더에 .vue, .scss 파일 생성 (PascalCase!)
