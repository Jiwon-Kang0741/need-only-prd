# 11. 공통코드 로딩

## 📋 이 단계의 목적

PFY 프로젝트의 **공통코드 로딩 패턴**을 이해하고 올바르게 구현합니다.

---

## 1️⃣ 기본 패턴

### 1.1 useCommonCodeStore 사용

```typescript
import { useCommonCodeStore } from '@/stores/commonCodeStore';
import { computed, ref, onMounted } from 'vue';

const common = useCommonCodeStore();
```

### 1.2 로딩 API 선택 (CRITICAL!)

PFY에서는 **두 가지 로딩 API**가 있습니다. 화면 특성에 따라 선택합니다.

| API | 용도 | 사용 빈도 |
|-----|------|----------|
| `ensureLoaded` | 코드 ID 배열로 간단 로딩 | ★★★ 가장 많음 |
| `ensureLoadedMulti` | Query 타입 + Code 타입 혼합 로딩 | ★★☆ |
| `loadMulti` | 동적 파라미터가 필요한 경우 | ★☆☆ |

---

## 2️⃣ ensureLoaded — 단순 코드 로딩 (가장 흔한 패턴)

코드 ID 배열만으로 로딩합니다. 이미 로딩된 코드는 건너뛰므로 중복 호출에 안전합니다.

```typescript
// spov010/index.vue
const common = useCommonCodeStore();
onMounted(async () => {
  await common.ensureLoaded(['COM_BIZ_0012'], 'Y');
});

// computed로 옵션 가져오기
const COM_BIZ_0012 = computed(() =>
  common.options('COM_BIZ_0012', true, 'ALL')
);
```

---

## 3️⃣ ensureLoadedMulti — Query 타입 + Code 타입 혼합 로딩

서버 쿼리 기반 공통코드와 단순 코드 타입을 한 번에 로딩합니다.

### 3.1 파라미터 구조

```typescript
// Query 타입 (서버 쿼리로 조회)
{
  id: `${windowId}_DATECOND`,         // 캐시 키 (options() 호출 시 동일 id 사용)
  queryId: 'MNPS010.DateCond',        // 서버 쿼리 ID
  whereClause: 'busnScIn=A1,; progId=PMDP030',  // 서버 필터 조건 (문자열)
}

// Code 타입 (공통코드 테이블 조회)
{
  id: 'COM_PBL_0005',                 // 코드 ID = 캐시 키
  useYn: 'Y',                         // 사용여부 필터
}
```

### 3.2 실제 사용 예시

```typescript
// pmdp030/index.vue
const windowId = 'PMDP030';
const common = useCommonCodeStore();

onMounted(async () => {
  const defaultWhereClause = `busnScIn=A1,; progId=${windowId}`;
  await common.ensureLoadedMulti([
    // Query 타입
    {
      id: `${windowId}_DATECOND`,
      queryId: 'MNPS010.DateCond',
      whereClause: defaultWhereClause,
    },
    {
      id: `${windowId}_STATCOMBOIN`,
      queryId: 'MNPS010.StatComboIn',
      whereClause: defaultWhereClause,
    },
    {
      id: `${windowId}_STRCOMBO`,
      queryId: 'MNOD090Query.StrCombo',
      whereClause: 'vndrCd=HSC01',
    },
    {
      id: `${windowId}_SUBSTRCOMBO_ALL`,
      queryId: 'MNOD090Query.SubStrCombo',
      whereClause: 'vndrCd=HSC01',
    },
    // Code 타입
    { id: 'COM_PBL_0005', useYn: 'Y' },
    { id: 'COM_PBL_0008', useYn: 'Y' },
  ]);
});
```

### 3.3 options() 호출 — id 기반

`ensureLoadedMulti`로 로딩한 데이터는 **`id` 값을 키**로 `options()`에서 가져옵니다.

```typescript
// Query 타입 — id로 접근, customParam 불필요 (서버에서 이미 필터링됨)
const DATECOND = computed(() =>
  common.options(`${windowId}_DATECOND`, true, '전체', 'ALL')
);
const STATCOMBOIN = computed(() =>
  common.options(`${windowId}_STATCOMBOIN`, true, '전체', 'ALL')
);
const strOptions = computed(() => {
  const opts = common.options(`${windowId}_STRCOMBO`, true, '선택', 'SELECT');
  return opts || [];
});

// Code 타입 — 클라이언트 필터링은 5번째 파라미터(whereClause)로 전달
const COM_PBL_0005 = computed(() =>
  common.options('COM_PBL_0005', false, '', '', 'pCodeCd=C')
);
const COM_PBL_0008 = computed(() =>
  common.options('COM_PBL_0008', false, '', '')
);
```

## 4️⃣ loadMulti — 동적 파라미터가 필요한 경우

Select 변경 시 다른 Select 옵션을 **서버에서 다시 로드**해야 할 때 사용합니다.

### 4.1 포맷 A: QueryId + customParam (PMDP010 패턴)

```typescript
// PMDP010SearchForm.vue — 차수 변경 → 부체계 동적 로드
const loadStrCdOptions = async (contNo: string) => {
  if (!contNo) {
    strCdOptions.value = [];
    return;
  }

  const param = `CONT_NO=${contNo}`;
  await commonCodeStore.loadMulti([
    { QueryId: 'MNOD090Query.StrCombo', customParam: param },
  ]);

  // ⭐ CRITICAL: options 호출 시에도 동일한 customParam 전달 (캐시 키 매칭)
  strCdOptions.value = commonCodeStore.options(
    'MNOD090Query.StrCombo',
    true,
    '전체',
    'ALL',
    undefined,
    param
  );
};
```

### 4.2 포맷 B: id + queryId + whereClause(객체) + reloadYn (MTPM 패턴)

이미 로딩된 데이터를 **강제 갱신**할 때 `reloadYn: 'Y'`를 사용합니다.

```typescript
// mtpm020p02/index.vue — 체계/부체계 변경 → 정비주기/계단 동적 로드
const loadDynamicCombos = async (pblCd: string, strCd: string) => {
  await common.loadMulti([
    {
      id: `${windowId}_RPAR_CYCLS_COMBO`,
      queryId: 'MTPM020Query.RparCyclsCombo',
      whereClause: { pblCd, strCd },
      reloadYn: 'Y',
    },
    {
      id: `${windowId}_RPAR_SC_COMBO`,
      queryId: 'MTPM020Query.RparScCombo',
      whereClause: { pblCd, strCd, bsCycl: '' },
      reloadYn: 'Y',
    },
  ]);
};
```

## 5️⃣ optionByWindowId — 화면별 코드 분기

동일 공통코드를 **화면마다 다른 옵션**으로 표시할 때 사용합니다.

```typescript
// spov010/index.vue
onMounted(async () => {
  await common.ensureLoaded(['COM_BIZ_0012'], 'Y');
  await common.optionByWindowId(
    ['COM_BIZ_0004', 'COM_BIZ_0009'],
    ['SPOV010', 'SPOV010']
  );
});
```

## 6️⃣ provide/inject 패턴 — 부모 로드 → 자식 주입

공통코드를 **부모(index.vue)**에서 로딩하고 **자식(SearchForm, DataTable)**에 `provide`하는 패턴입니다.

### 6.1 부모: index.vue

```typescript
// pmdp030/index.vue
// (공통코드 로딩 후)
provide(`${windowId}_STRCOMBO`, strOptions);
provide(`${windowId}_SUBSTRCOMBO_ALL`, subStrComboAll);
provide(`${windowId}_DATECOND`, DATECOND);
provide(`${windowId}_STATCOMBOIN`, STATCOMBOIN);
provide('COM_PBL_0005', COM_PBL_0005);
provide('COM_PBL_0008', COM_PBL_0008);
```

### 6.2 자식: SearchForm.vue

```typescript
// pmdp030SearchForm.vue
const windowId = inject('windowId') as string;

const strOptions = inject(`${windowId}_STRCOMBO`) as Ref<{ name: string; value: string }[]>;
const subStrComboAll = inject(`${windowId}_SUBSTRCOMBO_ALL`) as Ref<{ name: string; value: string }[]>;
const searchDtTypeOptions = inject(`${windowId}_DATECOND`) as Ref<{ name: string; value: string }[]>;
const prgStatOptions = inject(`${windowId}_STATCOMBOIN`) as any;
const COM_PBL_0005 = inject('COM_PBL_0005') as any;
```

**장점**: SearchForm에서 스토어를 직접 호출하지 않으므로 로딩 타이밍 문제가 없습니다.

## 7️⃣ computed로 옵션 정의 (CRITICAL!)

**⚠️ 반드시 `computed`로 감싸야 합니다. 직접 호출하면 초기 로딩 시 빈 배열이 반환됩니다.**

```typescript
// ✅ CORRECT — computed 사용 (반응형 업데이트)
const prgStatOptions = computed(() =>
  common.options('MNPS010Query.StatComboIn', true, '전체', 'ALL')
);

// ❌ WRONG — 직접 호출 (로딩 전 빈 배열 고정)
const prgStatOptions = common.options('MNPS010Query.StatComboIn', true, '전체', 'ALL');
```

## 8️⃣ options() 메서드 파라미터

### 8.1 메서드 시그니처

```typescript
common.options(
  idOrQueryId: string,         // 코드 ID 또는 ensureLoadedMulti의 id 값
  addAll: boolean = false,     // "전체" 옵션 추가 여부
  allLabel: string = '전체',    // "전체" 옵션 레이블 (화면에 따라 'ALL', '선택' 등)
  allValue: string = '',       // "전체" 옵션 값 (화면에 따라 '', 'ALL', 'SELECT' 등)
  whereClause?: string,        // 클라이언트 필터 조건 (예: 'pCodeCd=C')
  customParam?: string         // loadMulti 캐시 키 매칭용 (동적 로딩 시)
)
```

### 8.2 사용 예시

```typescript
// 1. 기본 사용 (전체 옵션 없음)
const ipgOptions = computed(() => common.options('COM_PBL_0010'));

// 2. 전체 옵션 포함
const prgStatOptions = computed(() =>
  common.options('MNPS010Query.StatComboIn', true, '전체', 'ALL')
);

// 3. 클라이언트 필터링 (whereClause)
const cancStatOptions = computed(() =>
  common.options('COM_PBL_0005', false, '', '', 'pCodeCd=C')
);

// 4. ensureLoadedMulti의 id 기반 접근
const DATECOND = computed(() =>
  common.options(`${windowId}_DATECOND`, true, '전체', 'ALL')
);

// 5. loadMulti 동적 로딩 후 customParam 매칭
const param = `CONT_NO=${contNo}`;
strCdOptions.value = common.options('MNOD090Query.StrCombo', true, '전체', 'ALL', undefined, param);
```

## 9️⃣ Select 컴포넌트 연결

### 9.1 optionLabel 속성 (CRITICAL!)

PFY에서 Select의 **optionLabel 속성**은 두 가지가 혼재합니다:

```vue
<!-- ✅ 주류 패턴 (대부분의 화면) — optionLabel="name" -->
<Select
  v-bind="props"
  :options="prgStatOptions"
  optionLabel="name"
  optionValue="value"
/>

<!-- ✅ PMDP010/040 패턴 — optionLabel="label" -->
<Select
  v-bind="props"
  :options="prgStatOptions"
  optionLabel="label"
  optionValue="value"
/>
```

**⚠️ 확실하지 않으면 `optionLabel="name"`을 사용하세요** (61개 파일 vs 11개 파일).

### 9.2 SearchFormField 안에서 사용

```vue
<SearchFormField v-slot="{ props, invalid }" name="prg_stat">
  <SearchFormLabel>진행상태</SearchFormLabel>
  <SearchFormContent>
    <Select
      v-bind="props"
      :invalid="invalid"
      :options="prgStatOptions"
      optionLabel="name"
      optionValue="value"
      placeholder="선택"
    />
  </SearchFormContent>
</SearchFormField>
```

### 9.3 동적 옵션 Select + @change 이벤트

```vue
<SearchFormField v-slot="{ props, invalid }" name="cont_no">
  <SearchFormLabel>차수</SearchFormLabel>
  <SearchFormContent>
    <Select
      v-bind="props"
      :invalid="invalid"
      :options="contNoOptions"
      optionLabel="name"
      optionValue="value"
      placeholder="선택"
      @change="onContNoChange"
    />
  </SearchFormContent>
</SearchFormField>
```

## 🔟 클라이언트 필터링 패턴

이미 로딩된 전체 옵션에서 **computed로 필터링**하여 하위 Select 옵션을 만드는 패턴입니다. 서버 재호출 없이 빠르게 동작합니다.

```typescript
// pmdp030SearchForm.vue — 체계 선택 → 부체계 필터링 (서버 재호출 없음)
const subStrOptions = computed(() => {
  const selected = searchParams.value?.pblCd;
  if (!selected || selected === 'SELECT') {
    return subStrComboAll.value || [];
  }
  return (subStrComboAll.value || []).filter(
    (item: any) => item.pblCd === selected
  );
});
```

## ✅ 체크리스트

### 공통코드 로딩
- [ ] `useCommonCodeStore` import
- [ ] `onMounted`에서 로딩 API 호출 (`ensureLoaded` / `ensureLoadedMulti` / `loadMulti`)
- [ ] **await** 사용하여 로딩 대기 (빠뜨리면 초기 빈 옵션)
- [ ] Query 타입 vs Code 타입 구분하여 파라미터 설정

### computed 옵션
- [ ] **반드시 `computed`로 옵션 정의** (직접 호출 금지)
- [ ] `options()` 파라미터 올바르게 전달 (id, addAll, allLabel, allValue, whereClause, customParam)

### 동적 로딩
- [ ] Select `@change` 이벤트 핸들러 구현
- [ ] `loadMulti` 호출 시 `customParam`/`whereClause` 전달
- [ ] **`options()` 호출 시에도 동일한 `customParam` 전달** (캐시 키 매칭!)

### Select 컴포넌트
- [ ] `v-bind="props"` 전달
- [ ] `:options` 바인딩 (computed ref)
- [ ] `optionLabel="name"` (주류 패턴) 또는 `optionLabel="label"`
- [ ] `optionValue="value"`

### provide/inject (해당 시)
- [ ] 부모(index.vue)에서 `provide` 설정
- [ ] 자식(SearchForm 등)에서 `inject` 접근
- [ ] inject 타입 캐스팅 (`as Ref<{ name: string; value: string }[]>`)
