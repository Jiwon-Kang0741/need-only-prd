## 📝 DTO 명명 규칙

### 중요: DTO 클래스명은 화면 코드를 붙이지 않음

tomms-lite에서는 **Service/DAO/Mapper는 화면 코드를 사용**하되, **DTO 클래스명은 화면 코드를 붙이지 않고 기능명만** 사용합니다.

**예시**:
- 화면 코드: `PMDP020` (부품요청)
- Service/DAO/Mapper: `PMDP020Service.java`, `PMDP020Dao.java`, `PMDP020Mapper.xml`
- DTO 클래스명: `SearchListReqDto`, `SaveResDto` (화면 코드 없음!)

**DTO 명명 패턴**:

| 기능 | Request DTO | Response DTO |
|-----|------------|-------------|
| 목록 조회 | `SearchListReqDto` | `SearchListResDto` |
| 상세 조회 | `SelectDetailReqDto` | `SelectDetailResDto` |
| 저장 | `SaveReqDto` | `SaveResDto` |
| 삭제 | `DeleteReqDto` | `DeleteResDto` |

**@ServiceId는 전체 화면 코드 사용**:
```java
// ✅ 올바른 예
@ServiceId("PMDP020/searchList")  // 전체 화면 코드
public List<SearchListResDto> searchList(SearchListReqDto request) {
    // ...
}
```

**파일 구조 예시**:
```
tomms-lite-war/tomms-biz-spareparts/
  src/main/java/hsc/tomms/web/pm/dp/
    ├── service/
    │   ├── PMDP020Service.java           ← 전체 화면 코드
    │   └── PMDP020ServiceImpl.java       ← 전체 화면 코드
    ├── dao/
    │   ├── PMDP020Dao.java               ← 전체 화면 코드
    │   └── PMDP020DaoImpl.java           ← 전체 화면 코드
    └── dto/
        ├── request/
        │   ├── SearchListReqDto.java      ← 화면 코드 없음!
        │   └── SaveReqDto.java            ← 화면 코드 없음!
        └── response/
            ├── SearchListResDto.java      ← 화면 코드 없음!
            └── SaveResDto.java            ← 화면 코드 없음!
```

**핵심 규칙**:
- ✅ Service/DAO/Mapper 파일명: 전체 화면 코드 사용 (예: `PMDP020Service.java`)
- ✅ DTO 클래스명: 화면 코드 없이 기능명만 사용 (예: `SearchListReqDto.java`)
- ✅ @ServiceId: 전체 화면 코드 사용 (예: `@ServiceId("PMDP020/searchList")`)
- ✅ 패키지 경로: 화면 코드의 모듈에 맞게 구성 (예: `hsc.tomms.web.pm.dp`)

### 1-2. 원본 코드 구조 파악

**확인 사항**:

1. **메서드 목록**
   - 조회 메서드: `selectList`, `selectDetail` 등
   - 등록 메서드: `insert`, `create` 등  
   - 수정 메서드: `update`, `modify` 등
   - 삭제 메서드: `delete`, `remove` 등

2. **입력 파라미터 분석**
   ```java
   // HTOMSS 원본 예시
   public RecordSet selectInventoryList(Record input) {
       String storageId = input.getString("storageId");
       String itemCode = input.getString("itemCode");
       String searchText = input.getString("searchText");
       int pageNo = input.getInt("pageNo", 1);
       int pageSize = input.getInt("pageSize", 10);
       // ...
   }
   ```
   
   **파라미터 목록 작성**:
   | 파라미터명 | 타입 | 필수여부 | 설명 |
   |-----------|------|---------|------|
   | storageId | String | Y | 창고ID |
   | itemCode | String | N | 품목코드 |
   | searchText | String | N | 검색어 |
   | pageNo | int | N | 페이지번호 (기본 1) |
   | pageSize | int | N | 페이지크기 (기본 10) |

3. **출력 데이터 분석**
   ```java
   // HTOMSS 원본 예시
   RecordSet result = dao.selectList(input);
   // RecordSet에 어떤 필드들이 들어있는지 파악
   ```
   
   **응답 필드 목록 작성**:
   | 필드명 | 타입 | 설명 |
   |-------|------|------|
   | invtId | String | 재고ID |
   | itemCode | String | 품목코드 |
   | itemName | String | 품목명 |
   | qty | int | 수량 |
   | unitPrice | BigDecimal | 단가 |

4. **Validation 로직 파악**
   ```java
   // ValidationMeta 사용 여부
   ValidationUtil.validate(input, ValidationMeta.builder()
       .required("storageId", "창고ID는 필수입니다")
       .build());
   ```

5. **비즈니스 로직 파악**
   - 단순 CRUD인지
   - 복잡한 계산 로직이 있는지
   - 외부 연동이 있는지
   - 트랜잭션 처리가 필요한지

## 📝 Step 3: DTO 구조 설계

### 3-1. Request DTO 설계

**예시: IMIR010 재고 조회 화면**

**파일명**: `010SearchListReqDto.java` (숫자 3자리 사용!)

```java
package hsc.tomms.web.im.ir.dto.request;

import lombok.Data;
import lombok.EqualsAndHashCode;
import hsc.tomms.web.common.dto.base.SearchBaseDto;

/**
 * <pre>
 * <li>SYSTEM      : TOMMS Lite
 * <li>BUSINESS    : 재고관리 - 재고조회
 * <li>FILE NAME   : 010SearchListReqDto / 재고목록조회요청DTO
 * <li>DESCRIPTION : 재고목록조회요청DTO
 * </pre>
 */
@Data
@EqualsAndHashCode(callSuper = false)
public class 010SearchListReqDto extends SearchBaseDto {
    
    /** 창고ID (필수) */
    private String storageId;
    
    /** 품목코드 (선택) */
    private String itemCode;
    
    /** 검색어 (선택) */
    private String searchText;
    
    /** 페이지번호 (기본 1) */
    private int pageNo = 1;
    
    /** 페이지크기 (기본 10) */
    private int pageSize = 10;
}
```

### 3-2. Response DTO 설계

**파일명**: `010SearchListResDto.java` (숫자 3자리 사용!)

```java
package hsc.tomms.web.im.ir.dto.response;

import java.math.BigDecimal;
import lombok.Data;

/**
 * <pre>
 * <li>SYSTEM      : TOMMS Lite
 * <li>BUSINESS    : 재고관리 - 재고조회
 * <li>FILE NAME   : 010SearchListResDto / 재고목록응답DTO
 * <li>DESCRIPTION : 재고목록응답DTO
 * </pre>
 */
@Data
public class 010SearchListResDto {
    
    /** 재고ID */
    private String invtId;
    
    /** 품목코드 */
    private String itemCode;
    
    /** 품목명 */
    private String itemName;
    
    /** 수량 */
    private int qty;
    
    /** 단가 */
    private BigDecimal unitPrice;
    
    /** 생성일시 */
    private String createDate;
}
```

### 3-3. DTO 설계 원칙

1. **Request DTO**
   - Record의 `getString()`, `getInt()` 등을 필드로 변환
   - 필수 필드는 주석으로 명시
   - 기본값이 있는 필드는 초기화

2. **Response DTO**
   - RecordSet의 컬럼을 필드로 변환
   - 날짜 타입은 String으로 (포맷팅 필요)
   - 금액 타입은 BigDecimal 사용

### 4-2. MyBatis Mapper 설계

**파일명**: `IMIR010Mapper.xml`  
**위치**: `src/main/resources/hsc/tomms/web/im/ir/mybatis/mappers/`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="hsc.tomms.web.im.ir.dao.IMIR010Dao">
    
    <!-- 재고 목록 조회 -->
    <select id="selectInventoryList" 
            parameterType="hsc.tomms.web.im.ir.dto.request.InventoryListReqDto"
            resultType="hsc.tomms.web.im.ir.dto.response.InventoryListResDto">
        SELECT 
            INVT_ID as invtId,
            ITEM_CODE as itemCode,
            ITEM_NAME as itemName,
            QTY as qty,
            UNIT_PRICE as unitPrice,
            CREATE_DATE as createDate
        FROM TB_INVENTORY
        WHERE 1=1
          AND STORAGE_ID = #{storageId}
          <if test="itemCode != null and itemCode != ''">
          AND ITEM_CODE = #{itemCode}
          </if>
          <if test="searchText != null and searchText != ''">
          AND (ITEM_CODE LIKE '%' || #{searchText} || '%' 
               OR ITEM_NAME LIKE '%' || #{searchText} || '%')
          </if>
        ORDER BY CREATE_DATE DESC
        LIMIT #{pageSize} OFFSET #{pageNo}
    </select>
    
</mapper>
```

### 4-3. HQML → MyBatis Mapper 변환

**동적 쿼리 태그 변환**:

| HQML | MyBatis |
|------|---------|
| `<isNotEmpty>` | `<if test="param != null and param != ''">` |
| `<isEmpty>` | `<if test="param == null or param == ''">` |
| `<isEqual>` | `<if test="param == 'value'">` |

## ❓ 문제 해결

### Q1: 원본 Record 파라미터를 어떻게 DTO로 변환하나요?

**답변**:
- Record의 `getString("key")` → DTO 필드 `private String key`
- Record의 `getInt("key")` → DTO 필드 `private int key`
- Record의 `get객체타입("key")` → 해당 타입으로 변환

### Q2: RecordSet 응답을 어떻게 DTO로 변환하나요?

**답변**:
- HQML 또는 DAO의 select 쿼리 결과 컬럼을 확인
- 각 컬럼을 Response DTO 필드로 매핑
- MyBatis의 resultType에 Response DTO 지정

### Q3: 동적 쿼리 태그를 MyBatis로 어떻게 변환하나요?

**답변**:
- HQML의 `<isNotEmpty property="key">` → `<if test="key != null and key != ''">`
- HQML의 `<isEqual property="key" compareValue="value">` → `<if test="key == 'value'">`
