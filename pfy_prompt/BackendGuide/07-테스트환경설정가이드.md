## 개요

tomms-lite 프로젝트는 커스텀 프레임워크(HONE Framework)를 사용하므로, Spring Boot 테스트를 실행하기 위해서는 다음과 같은 설정이 필요합니다:

- **Maven 의존성**: JUnit 5 및 Spring Boot Test 의존성
- **테스트 설정 클래스**: Spring Boot 테스트 컨텍스트 설정
- **리소스 파일**: DataSource, MyBatis, Application 설정 파일

### ⚠️ 공통 리소스 사용 방법

**대부분의 설정 파일은 모든 모듈에서 동일하므로, 공통 리소스를 사용하는 것을 권장합니다.**

**공통 리소스 위치**: `tomms-boot/src/test/resources/`

공통으로 사용 가능한 파일:
- ✅ `application.yml` - 모든 모듈 동일
- ✅ `application-test.yml` - 모든 모듈 동일
- ✅ `datasources/datasources-main-test.yml` - 데이터베이스 연결 정보 동일
- ✅ `datasources/datasources-main-local.yml` - 데이터베이스 연결 정보 동일
- ✅ `honeconfig/mybatis/configuration.xml` - MyBatis 설정 완전히 동일
- ✅ `logback-test.xml` - 로깅 설정 거의 동일

모듈별로 생성해야 하는 파일:
- ⚠️ `TestConfiguration.java` - ComponentScan 패키지 경로가 모듈별로 다름

**공통 리소스 사용 방법**:
1. `tomms-boot/src/test/resources/`에 공통 리소스 파일 생성
2. 각 모듈의 테스트에서 `tomms-boot`를 test dependency로 추가 (이미 포함되어 있음)
3. 각 모듈은 `TestConfiguration.java`만 생성

**또는 각 모듈별로 생성**:
- 모듈별로 다른 설정이 필요한 경우에만 각 모듈에 생성

## 필수 설정 파일 목록

각 모듈(`tomms-biz-{module}`)에서 테스트를 실행하려면 다음 파일들이 필요합니다:

### 방법 1: 공통 리소스 사용 (권장) ⭐

**공통 리소스 위치**: `tomms-boot/src/test/resources/`

```
tomms-boot/
  src/test/resources/
    ├── application.yml  (공통)
    ├── application-test.yml  (공통)
    ├── logback-test.xml  (공통)
    ├── datasources/
    │   ├── datasources-main-test.yml  (공통)
    │   └── datasources-main-local.yml  (공통)
    └── honeconfig/
        └── mybatis/
            └── configuration.xml  (공통)
```

**각 모듈별로 생성**:
```
tomms-biz-{module}/
  ├── pom.xml  (수정 필요)
  └── src/test/java/
      hsc/tomms/web/{module}/{category}/provider/
        └── TestConfiguration.java  (신규 생성)
```

**장점**:
- ✅ 설정 파일 중복 제거
- ✅ 일관된 설정 관리
- ✅ 설정 변경 시 한 곳만 수정

### 방법 2: 각 모듈별로 생성

**각 모듈에 모든 파일 생성**:

```
tomms-biz-{module}/
  ├── pom.xml  (수정 필요)
  ├── src/test/java/
  │   hsc/tomms/web/{module}/{category}/provider/
  │     └── TestConfiguration.java  (신규 생성)
  └── src/test/resources/
      ├── application.yml  (신규 생성)
      ├── application-test.yml  (신규 생성)
      ├── logback-test.xml  (신규 생성)
      ├── datasources/
      │   ├── datasources-main-test.yml  (신규 생성)
      │   └── datasources-main-local.yml  (신규 생성)
      └── honeconfig/
          └── mybatis/
              └── configuration.xml  (신규 생성)
```

**장점**:
- ✅ 모듈별로 다른 설정 가능
- ✅ 모듈 독립성

**단점**:
- ❌ 설정 파일 중복
- ❌ 설정 변경 시 여러 곳 수정 필요

## 단계별 설정 가이드

### 방법 선택: 공통 리소스 vs 모듈별 리소스

**권장**: 공통 리소스 사용 (방법 1)
- 대부분의 설정이 동일하므로 공통 리소스를 사용하는 것이 효율적입니다.
- `tomms-boot/src/test/resources/`에 공통 리소스를 생성하고, 각 모듈은 `TestConfiguration.java`만 생성합니다.

**선택**: 모듈별 리소스 (방법 2)
- 모듈별로 다른 설정이 필요한 경우에만 각 모듈에 리소스를 생성합니다.

---

## 방법 1: 공통 리소스 사용 (권장) ⭐

### Step 1-1: tomms-boot에 공통 리소스 생성

**위치**: `tomms-boot/src/test/resources/`

다음 파일들을 `tomms-boot/src/test/resources/`에 생성합니다:
- `application.yml`
- `application-test.yml`
- `logback-test.xml`
- `datasources/datasources-main-test.yml`
- `datasources/datasources-main-local.yml`
- `honeconfig/mybatis/configuration.xml`

**참고**: 각 파일 내용은 아래 "방법 2"의 Step 3~8과 동일합니다.

### Step 1-2: 각 모듈의 pom.xml 수정

**위치**: `tomms-biz-{module}/pom.xml`

`tomms-boot`가 이미 dependency로 포함되어 있으므로 추가 작업 불필요합니다.

### Step 1-3: 각 모듈에 TestConfiguration.java 생성

**위치**: `tomms-biz-{module}/src/test/java/hsc/tomms/web/{module}/{category}/provider/TestConfiguration.java`

각 모듈별로 `TestConfiguration.java`만 생성하면 됩니다. (Step 2 참고)

### Step 2: TestConfiguration.java 생성

**위치**: `src/test/java/hsc/tomms/web/{module}/{category}/provider/TestConfiguration.java`

**템플릿**:

```java
/*
 * You are strictly prohibited to copy, disclose, distribute, modify, or use this program in part 
 * or as a whole without the prior written consent of Hanwha Systems Co., Ltd.
 * Hanwha Systems Co., Ltd., owns the intellectual property rights in and to this program.

package hsc.tomms.web.{module}.{category}.provider;

import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration;
import org.springframework.boot.autoconfigure.jdbc.DataSourceTransactionManagerAutoConfiguration;
import org.springframework.boot.autoconfigure.orm.jpa.HibernateJpaAutoConfiguration;
import org.springframework.boot.autoconfigure.sql.init.SqlInitializationAutoConfiguration;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Import;

import hone.bom.dao.datasource.DataSourceNameFinder;
import hone.bom.config.HoneConfigurationConstants;
import hone.bom.web.util.servlet.ServletContextInitializerUtils;
import hone.bom.dao.datasource.config.DynamicDataSourceConfig;
import hone.bom.yaml.YamlBasedPropertiesConfig;
import javax.annotation.PostConstruct;

/**
 * <pre>
 * <li>SYSTEM      : TOMMS Lite
 * <li>BUSINESS    : {업무명}
 * <li>FILE NAME   : TestConfiguration / 테스트용 Spring Boot 설정 클래스
 * <li>DESCRIPTION : Provider 테스트를 위한 최소 Spring Boot 설정
 * ----
 * 2025.01.07	개발자	최초작성
 * ----
 * </pre>
 */
@SpringBootApplication(exclude = {
    DataSourceAutoConfiguration.class,
    DataSourceTransactionManagerAutoConfiguration.class,
    HibernateJpaAutoConfiguration.class,
    SqlInitializationAutoConfiguration.class
})
@Import({
    YamlBasedPropertiesConfig.class,
    DynamicDataSourceConfig.class
})
@ComponentScan(basePackages = {
    "hsc.tomms.web.{module}.{category}.provider",
    "hsc.tomms.web.{module}.{category}.dao",
    "hsc.tomms.web.{module}.{category}.service"
})
@Configuration
public class TestConfiguration {
    
    static {
        // 프레임워크 구동 모드를 'boot'로 설정
        System.setProperty(HoneConfigurationConstants.STARTUP_MODE_KEY, "boot");
        
        // 빈 정의 오버라이딩 허용
        System.setProperty("spring.main.allow-bean-definition-overriding", "true");
        
        // 시스템 YML 파일 위치 설정 (테스트 리소스 경로)
        String location = System.getProperty(
            HoneConfigurationConstants.APPLICATION_YML_LOCATION_KEY,
            "classpath:application-test.yml"
        );
        
        // 시스템 속성 초기화 (프로필 설정 포함)
        // 이 초기화가 PropertyServiceHolder를 초기화합니다
        ServletContextInitializerUtils.initSystemProperties(
            location, 
            HoneConfigurationConstants.PROFILE_KEY
        );
    }
    
    /**
     * PropertyServiceHolder 초기화를 보장하기 위한 @PostConstruct 메서드
     * Spring 컨텍스트 초기화 후에 실행됩니다.
     */
    @PostConstruct
    public void ensurePropertyServiceHolderInitialized() {
        // PropertyServiceHolder가 초기화되었는지 확인
        // 이미 정적 초기화 블록에서 초기화되었지만, 
        // Spring 컨텍스트 초기화 시점에 다시 확인합니다.
        try {
            hone.bom.commons.property.PropertyServiceHolder.getAll();
        } catch (Exception e) {
            // PropertyServiceHolder가 초기화되지 않은 경우
            // 시스템 속성을 다시 초기화합니다.
            String location = System.getProperty(
                HoneConfigurationConstants.APPLICATION_YML_LOCATION_KEY,
                "classpath:application-test.yml"
            );
            ServletContextInitializerUtils.initSystemProperties(
                location, 
                HoneConfigurationConstants.PROFILE_KEY
            );
        }
    }
    
    /**
     * DataSourceNameFinder 빈 생성
     * PropertyServiceHolder 초기화 문제를 우회하기 위해 간단한 구현을 제공합니다.
     */
    @Bean
    public DataSourceNameFinder dataSourceNameFinder() {
        // PropertyServiceHolder 초기화 문제를 우회하기 위해
        // 간단한 DataSourceNameFinder 구현을 제공합니다.
        return new DataSourceNameFinder() {
            @Override
            public String findByClassName(String className) {
                // 기본 DataSource 이름 반환
                return "dataSource";
            }
            
            @Override
            public String find(hone.bom.dao.datasource.DataSourceLookupDao dao) {
                // 기본 DataSource 이름 반환
                return "dataSource";
            }
        };
    }
}
```

**중요**: 
- `{module}`, `{category}` 부분을 실제 모듈명과 카테고리명으로 변경하세요.
- `ComponentScan`의 `basePackages`를 테스트할 패키지 경로로 수정하세요.

**참고**: `tomms-biz-master/src/test/java/hsc/tomms/web/mn/od/provider/TestConfiguration.java`를 참고하세요.

```xml
<configuration>
    <!-- 테스트 실행 시 콘솔에 로그 출력 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <!-- 테스트 클래스 로그 레벨 -->
    <logger name="hsc.tomms.web" level="DEBUG" />
    
    <!-- Spring Framework 로그 레벨 -->
    <logger name="org.springframework" level="INFO" />
    
    <!-- HONE Framework 로그 레벨 -->
    <logger name="hone" level="WARN" />
    
    <!-- MyBatis 로그 레벨 -->
    <logger name="org.apache.ibatis" level="DEBUG" />
    <logger name="java.sql" level="DEBUG" />
    <logger name="java.sql.Connection" level="DEBUG" />
    <logger name="java.sql.Statement" level="DEBUG" />
    <logger name="java.sql.PreparedStatement" level="DEBUG" />
    <logger name="java.sql.ResultSet" level="DEBUG" />
    
    <!-- 루트 로거 설정 -->
    <root level="INFO">
        <appender-ref ref="CONSOLE" />
    </root>
</configuration>
```

**중요**: 
- 이 파일은 IDE에서 테스트를 실행할 때 로그 출력을 표시하기 위한 것입니다.
- `logback-test.xml`은 테스트 실행 시 자동으로 로드됩니다.

**참고**: `tomms-biz-master/src/test/resources/logback-test.xml`을 참고하세요.

## 테스트 실행 방법

### 1. 터미널에서 실행

#### 전체 테스트 실행

```bash
cd tomms-biz-{module}
mvn test
```

#### 특정 테스트 클래스 실행

```bash
cd tomms-biz-{module}
mvn test -Dtest=ClassNameTest
```

#### 특정 테스트 메서드 실행

```bash
cd tomms-biz-{module}
mvn test -Dtest='ClassNameTest#methodName'
```

### 2. IDE에서 실행 (Cursor/VS Code)

#### 테스트 출력 표시 설정

1. **Java Extension Pack 설정 확인**:
   - `File` → `Preferences` → `Settings` (또는 `Cmd+,`)
   - 검색: `java.test`
   - `Java: Test Config` 설정 확인

2. **테스트 실행 방법**:
   - 테스트 클래스 또는 메서드 위에 나타나는 "Run Test" 버튼 클릭
   - 또는 `Ctrl+Shift+P` → `Java: Run Tests`

3. **출력 확인**:
   - 테스트 실행 후 하단의 "Test Results" 패널 확인
   - "Output" 탭에서 로그 출력 확인
   - 출력이 보이지 않으면 `View` → `Output` → `Java Test Runner` 선택

#### 문제 해결

**"테스트 실행에서 출력을 기록하지 않았습니다" 메시지가 나오는 경우**:

1. **pom.xml 설정 확인**:
   - `maven-surefire-plugin`의 `configuration` 섹션이 올바르게 설정되었는지 확인
   - `redirectTestOutputToFile`이 `false`로 설정되어 있는지 확인

2. **logback-test.xml 확인**:
   - `src/test/resources/logback-test.xml` 파일이 존재하는지 확인
   - 콘솔 appender가 올바르게 설정되었는지 확인

3. **IDE 재시작**:
   - Cursor IDE를 재시작하여 설정 적용

4. **터미널에서 테스트 실행**:
   - IDE 대신 터미널에서 `mvn test` 실행하여 출력 확인

### 3. IntelliJ IDEA에서 실행

1. **테스트 실행**:
   - 테스트 클래스 또는 메서드 옆의 실행 버튼 클릭
   - 또는 `Ctrl+Shift+F10` (Windows/Linux) / `Ctrl+Shift+R` (Mac)

2. **출력 확인**:
   - 하단의 "Run" 탭에서 테스트 결과 확인
   - "Console" 탭에서 로그 출력 확인

3. **설정 확인**:
   - `File` → `Settings` → `Build, Execution, Deployment` → `Build Tools` → `Maven` → `Runner`
   - "Delegate IDE build/run actions to Maven" 옵션 확인

## 문제 해결

### 문제 1: "Could not resolve dependencies" 에러

**원인**: Maven 저장소 설정 문제

**해결**:
1. `~/.m2/settings.xml` 확인
2. `tomms-public` 저장소가 올바르게 설정되어 있는지 확인
3. `mvn clean install` 실행

**참고**: `maven/config/settings.xml`을 참고하세요.

---

### 문제 2: "Failed to load ApplicationContext" 에러

**원인**: 테스트 리소스 파일 누락

**해결**:
1. 다음 파일들이 모두 존재하는지 확인:
   - `src/test/resources/application.yml`
   - `src/test/resources/application-test.yml`
   - `src/test/resources/datasources/datasources-main-test.yml`
   - `src/test/resources/datasources/datasources-main-local.yml`
   - `src/test/resources/honeconfig/mybatis/configuration.xml`

2. `TestConfiguration.java`의 `ComponentScan` 패키지 경로 확인

---

### 문제 3: "데이타소스(dataSource) 연결정보가 없습니다" 에러

**원인**: DataSource 설정 파일 누락 또는 경로 오류

**원인**: `TestConfiguration`의 static 초기화 블록이 실행되지 않음

   ```xml
     <useSystemClassLoader>false</useSystemClassLoader>
     <reportFormat>brief</reportFormat>
   </configuration>
   ```

2. `src/test/resources/logback-test.xml` 파일 생성 확인

   - Cursor IDE: 테스트 실행 설정에서 "Show test output" 옵션 활성화
   - IntelliJ IDEA: Run/Debug Configurations에서 "Redirect output to console" 옵션 확인

4. 터미널에서 테스트 실행하여 출력 확인:
   ```bash
   cd tomms-biz-{module}
   mvn test -Dtest=ClassNameTest
   ```

### 문제 7: "JUnit5TestLoader not found" 또는 "NoClassDefFoundError: LauncherFactory" (IDE에서)

**원인**: IDE의 Eclipse JDT 러너가 JUnit Platform Launcher를 클래스패스에서 찾지 못함

**해결** (폐쇄망 환경):
1. **`spring-boot-starter-test`만 사용** (권장):
   ```xml
   <dependencies>
       <!-- spring-boot-starter-test에 이미 JUnit 5가 포함되어 있음 -->
       <!-- Spring Boot 2.7.18은 JUnit 5.8.2를 포함하고 있음 -->
       <dependency>
           <groupId>org.springframework.boot</groupId>
           <artifactId>spring-boot-starter-test</artifactId>
           <version>2.7.18</version>
           <scope>test</scope>
           <exclusions>
               <!-- JUnit 4 제외 (JUnit 5만 사용) -->
               <exclusion>
                   <groupId>org.junit.vintage</groupId>
                   <artifactId>junit-vintage-engine</artifactId>
               </exclusion>
           </exclusions>
       </dependency>
   </dependencies>
   ```

2. Maven Surefire 플러그인 설정:
   ```xml
   <plugin>
       <groupId>org.apache.maven.plugins</groupId>
       <artifactId>maven-surefire-plugin</artifactId>
       <version>3.1.2</version>
       <configuration>
           <printSummary>true</printSummary>
           <redirectTestOutputToFile>false</redirectTestOutputToFile>
           <useSystemClassLoader>false</useSystemClassLoader>
           <reportFormat>brief</reportFormat>
       </configuration>
       <dependencies>
           <!-- JUnit Platform 지원 -->
           <!-- spring-boot-starter-test에 포함된 JUnit 버전 사용 -->
           <dependency>
               <groupId>org.apache.maven.surefire</groupId>
               <artifactId>surefire-junit-platform</artifactId>
               <version>3.1.2</version>
           </dependency>
       </dependencies>
   </plugin>
   ```

3. Maven 프로젝트 새로고침:
   - Cursor IDE: `Ctrl+Shift+P` → `Java: Clean Java Language Server Workspace`
   - 또는 터미널에서: `mvn clean install -DskipTests`

4. IDE 재시작

**참고**: 
- **폐쇄망 환경에서는 명시적 JUnit 의존성을 추가하지 않는 것이 좋습니다.**
- `spring-boot-starter-test`에 이미 JUnit 5.8.2가 포함되어 있어 별도 의존성 추가가 불필요합니다.
- IDE가 `spring-boot-starter-test`의 의존성을 인식하면 자동으로 JUnit Platform Launcher도 포함됩니다.
- 만약 IDE에서 여전히 문제가 발생하면, IDE의 Maven 프로젝트를 완전히 새로고침하고 IDE를 재시작하세요.

## 필수 파일 vs 선택 파일

### ✅ 필수 파일 (반드시 필요)

1. **`TestConfiguration.java`** - 필수
   - Spring Boot 테스트 컨텍스트 설정
   - 없으면 테스트 실행 불가

2. **`application-test.yml`** - 필수
   - `TestConfiguration.java`에서 명시적으로 참조 (`classpath:application-test.yml`)
   - 없으면 PropertyServiceHolder 초기화 실패

3. **`datasources/datasources-main-test.yml`** - 필수
   - DataSource 설정 파일
   - 없으면 DataSource 연결 실패

4. **`honeconfig/mybatis/configuration.xml`** - 필수
   - MyBatis 설정 파일
   - 없으면 MyBatis 초기화 실패

### ⚠️ 선택 파일 (없어도 동작하지만 권장)

5. **`datasources/datasources-main-local.yml`** - 선택 (권장)
   - 프레임워크가 특정 조건에서 이 파일을 찾을 수 있음
   - `datasources-main-test.yml`과 동일한 내용이지만, 프레임워크가 자동으로 찾을 수 있도록 생성 권장
   - **내용**: `datasources-main-test.yml`과 동일하게 복사

6. **`logback-test.xml`** - 선택 (IDE 출력용)
   - IDE에서 테스트 실행 시 로그 출력을 보기 위한 파일
   - 없어도 테스트는 실행되지만, IDE에서 로그가 보이지 않을 수 있음
   - 터미널에서 `mvn test` 실행 시에는 필요 없음

### ❓ 불필요할 수 있는 파일

7. **`application.yml`** - 불필요할 수 있음
   - `@ActiveProfiles("test")`를 사용하면 `application-test.yml`이 우선됨
   - `application-test.yml`만 있어도 테스트 실행 가능
   - **하지만**: Spring Boot는 기본적으로 `application.yml`을 먼저 로드하므로, 공통 설정을 위해 유지하는 것을 권장

### 📋 최소 필수 파일 목록

**최소한으로 테스트를 실행하려면**:
```
tomms-biz-{module}/
  ├── pom.xml (수정)
  ├── src/test/java/.../TestConfiguration.java  ✅ 필수
  └── src/test/resources/
      ├── application-test.yml  ✅ 필수
      ├── datasources/
      │   └── datasources-main-test.yml  ✅ 필수
      └── honeconfig/mybatis/
          └── configuration.xml  ✅ 필수
```

**권장 파일 (IDE 사용 시)**:
```
tomms-biz-{module}/
  └── src/test/resources/
      ├── application.yml  ⚠️ 권장 (공통 설정)
      ├── application-test.yml  ✅ 필수
      ├── logback-test.xml  ⚠️ 권장 (IDE 출력용)
      ├── datasources/
      │   ├── datasources-main-test.yml  ✅ 필수
      │   └── datasources-main-local.yml  ⚠️ 권장 (프레임워크 호환성)
      └── honeconfig/mybatis/
          └── configuration.xml  ✅ 필수
```
