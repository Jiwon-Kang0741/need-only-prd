# 코드 변환 가이드

**목적**: HTOMSS 화면 코드를 tomms-lite 스타일로 변환하는 패턴 가이드

---

## 📋 변환 개요

### 주요 변환 항목

1. **데이터 타입**: Record/RecordSet → Request/Response DTO
2. **Service 어노테이션**: @ServiceId, @ServiceName 추가
3. **DAO**: Interface + Impl 패턴
4. **MyBatis Mapper**: HQML → XML Mapper
5. **Validation**: 입력값 검증 로직
6. **Transaction**: @Transactional 적용

---

**tomms-lite 변환**:
```java
package hsc.tomms.web.im.ir.service;

import has.fw.config.annotation.ServiceId;
import has.fw.config.annotation.ServiceName;
import hsc.tomms.web.im.ir.dao.IMIR010Dao;
import hsc.tomms.web.im.ir.dto.request.InventoryListReqDto;
import hsc.tomms.web.im.ir.dto.response.InventoryListResDto;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import java.util.List;

@Slf4j
@Service("IMIR010Service-재고관리")
@RequiredArgsConstructor
public class IMIR010ServiceImpl implements IMIR010Service {
    
    private final IMIR010Dao imir010Dao;
    
    @Override
    @ServiceId("IMIR010/selectInventoryList")
    @ServiceName("재고 목록 조회")
    public List<InventoryListResDto> selectInventoryList(InventoryListReqDto request) {
        log.info("재고 목록 조회 시작: {}", request);
        
        // Validation
        if (request.getStorageId() == null || request.getStorageId().isEmpty()) {
            throw new IllegalArgumentException("창고ID는 필수입니다.");
        }
        
        // DAO 호출 (DTO 전달)
        List<InventoryListResDto> result = imir010Dao.selectInventoryList(request);
        
        log.info("재고 목록 조회 완료: {} 건", result.size());
        return result;
    }
}
```

**변환 포인트**:
1. ✅ `Record input` → `InventoryListReqDto request`
2. ✅ `RecordSet` 반환 → `List<InventoryListResDto>` 반환
3. ✅ `@ServiceId`, `@ServiceName` 어노테이션 추가
4. ✅ `@RequiredArgsConstructor` + `private final` 생성자 주입
5. ✅ `@Slf4j` + `log.info()` 로깅 추가
6. ✅ Record에서 값 추출 불필요 (DTO getter 사용)

```java
@Override
public Record saveInventory(Record input) {
    String invtId = input.getString("invtId");
    String itemCode = input.getString("itemCode");
```

**tomms-lite 변환**:
```java
@Override
@Transactional
@ServiceId("IMIR010/saveInventory")
@ServiceName("재고 저장")
public Map<String, Object> saveInventory(InventorySaveReqDto request) {
    log.info("재고 저장 시작: {}", request);
    
    // Validation
    validateInventorySave(request);
    
    int result = 0;
    if (request.getInvtId() == null || request.getInvtId().isEmpty()) {
        // 등록
        result = imir010Dao.insertInventory(request);
    } else {
        // 수정
        result = imir010Dao.updateInventory(request);
    }
    
    Map<String, Object> response = new HashMap<>();
    response.put("success", result > 0);
    response.put("message", result > 0 ? "저장되었습니다." : "저장 실패");
    
    log.info("재고 저장 완료: {}", response);
    return response;
}

private void validateInventorySave(InventorySaveReqDto request) {
    if (request.getItemCode() == null || request.getItemCode().isEmpty()) {
        throw new IllegalArgumentException("품목코드는 필수입니다.");
    }
    if (request.getStorageId() == null || request.getStorageId().isEmpty()) {
        throw new IllegalArgumentException("창고ID는 필수입니다.");
    }
    if (request.getQty() < 0) {
        throw new IllegalArgumentException("수량은 0 이상이어야 합니다.");
    }
}
```

**변환 포인트**:
1. ✅ `@Transactional` 추가 (CUD 작업)
2. ✅ `Record` 반환 → `Map<String, Object>` 반환
3. ✅ Validation 로직을 별도 private 메서드로 분리
4. ✅ Record.set() → Map.put()

### 3-2. 동적 쿼리 태그 변환

**HTOMSS 원본**:
```xml

    <isNotEmpty property="searchText">

    </isNotEmpty>
    <isEqual property="statusCode" compareValue="ACTIVE">

    </isEqual>
    <isGreaterThan property="qty" compareValue="0">
        AND QTY > #qty#
    </isGreaterThan>

```

**tomms-lite 변환**:
```xml
<select id="selectInventoryList" 
        parameterType="hsc.tomms.web.im.ir.dto.request.InventoryListReqDto"
        resultType="hsc.tomms.web.im.ir.dto.response.InventoryListResDto">
    SELECT * FROM TB_INVENTORY
    WHERE 1=1
    <if test="itemCode != null and itemCode != ''">

    </if>
```

### 3-3. 동적 쿼리 태그 변환표

| HTOMSS (HQML) | tomms-lite (MyBatis) |
|---------------|----------------------|
| `<isNotEmpty property="field">` | `<if test="field != null and field != ''">` |
| `<isEmpty property="field">` | `<if test="field == null or field == ''">` |
| `<isEqual property="field" compareValue="value">` | `<if test="field == 'value'">` |
| `<isNotEqual property="field" compareValue="value">` | `<if test="field != 'value'">` |
| `<isGreaterThan property="field" compareValue="10">` | `<if test="field > 10">` |
| `<isLessThan property="field" compareValue="10">` | `<if test="field < 10">` |
| `#field#` | `#{field}` |
| `$field$` | `${field}` |

### 3-4. 페이징 쿼리 변환 (Oracle → MariaDB)

**HTOMSS 원본 (Oracle)**:
```xml
<select id="selectInventoryList">
    SELECT * FROM (
        SELECT ROWNUM AS RNUM, A.* FROM (
            SELECT * FROM TB_INVENTORY
            WHERE STORAGE_ID = #storageId#
            ORDER BY CREATE_DATE DESC
        ) A
        WHERE ROWNUM <= (#pageNo# * #pageSize#)
    )
    WHERE RNUM > ((#pageNo# - 1) * #pageSize#)
</select>
```

**tomms-lite 변환 (MariaDB)**:
```xml
<select id="selectInventoryList" 
        parameterType="hsc.tomms.web.im.ir.dto.request.InventoryListReqDto"
        resultType="hsc.tomms.web.im.ir.dto.response.InventoryListResDto">
    SELECT * FROM TB_INVENTORY
    WHERE STORAGE_ID = #{storageId}
    ORDER BY CREATE_DATE DESC
    LIMIT #{pageSize} OFFSET #{offset}
</select>
```

**변환 포인트**:
1. ✅ Oracle ROWNUM → MariaDB `LIMIT ... OFFSET ...` 사용
2. ✅ offset 계산: `(pageNo - 1) * pageSize`
3. ✅ 중첩 쿼리 제거하여 가독성 향상

### 3-5. Oracle → MariaDB SQL 함수 변환표

**중요**: HTOMSS는 Oracle 기반이지만, tomms-lite는 **MariaDB 기반**입니다.

| 기능 | Oracle (HTOMSS) | MariaDB (tomms-lite) | 비고 |
|------|----------------|---------------------|------|
| **날짜 포맷** | `TO_CHAR(date, 'YYYY-MM-DD')` | `DATE_FORMAT(date, '%Y-%m-%d')` | 포맷 문자열 다름 |
| **문자열 결합** | `'A' \|\| 'B'` 또는 `CONCAT('A', 'B')` | `CONCAT('A', 'B')` | `\|\|` 사용 불가 |
| **LIKE 검색** | `LIKE '%' \|\| #{text} \|\| '%'` | `LIKE CONCAT('%', #{text}, '%')` | CONCAT 필수 |
| **현재 날짜** | `SYSDATE` | `NOW()` 또는 `CURRENT_TIMESTAMP` | |
| **NULL 처리** | `NVL(field, default)` | `IFNULL(field, default)` | |
| **조건식** | `DECODE(field, 'A', 1, 2)` | `CASE WHEN field='A' THEN 1 ELSE 2 END` | DECODE 없음 |
| **시퀀스** | `SEQ_NAME.NEXTVAL` | `AUTO_INCREMENT` 또는 별도 시퀀스 테이블 | MariaDB 10.3+ 시퀀스 지원 |
| **페이징** | `ROWNUM`, `ROW_NUMBER()` | `LIMIT #{size} OFFSET #{offset}` | |
| **문자열 자르기** | `SUBSTR(field, 1, 10)` | `SUBSTRING(field, 1, 10)` | 동일하게 동작 |
| **날짜 연산** | `ADD_MONTHS(date, 1)` | `DATE_ADD(date, INTERVAL 1 MONTH)` | |
| **듀얼 테이블** | `SELECT 1 FROM DUAL` | `SELECT 1` | DUAL 불필요 |

### 3-6. Oracle → MariaDB 변환 예시

**예시 1: 날짜 포맷 + 문자열 결합**

```xml
<!-- Oracle (HTOMSS) -->
<select id="selectList">
    SELECT 
        INVT_ID,
        TO_CHAR(CREATE_DATE, 'YYYY-MM-DD HH24:MI:SS') AS CREATE_DATE,
        'INV-' || INVT_ID AS FULL_ID
    FROM TB_INVENTORY
    WHERE ITEM_NAME LIKE '%' || #{searchText} || '%'
</select>

<!-- MariaDB (tomms-lite) -->
<select id="selectList">
    SELECT 
        INVT_ID,
        DATE_FORMAT(CREATE_DATE, '%Y-%m-%d %H:%i:%s') AS CREATE_DATE,
        CONCAT('INV-', INVT_ID) AS FULL_ID
    FROM TB_INVENTORY
    WHERE ITEM_NAME LIKE CONCAT('%', #{searchText}, '%')
</select>
```

**예시 2: NULL 처리 + 조건식**

```xml
<!-- Oracle (HTOMSS) -->
<select id="selectList">
    SELECT 
        INVT_ID,
        NVL(QTY, 0) AS QTY,
        DECODE(STATUS_CODE, 'ACTIVE', '활성', '비활성') AS STATUS_NAME
    FROM TB_INVENTORY
</select>

<!-- MariaDB (tomms-lite) -->
<select id="selectList">
    SELECT 
        INVT_ID,
        IFNULL(QTY, 0) AS QTY,
        CASE WHEN STATUS_CODE = 'ACTIVE' THEN '활성' ELSE '비활성' END AS STATUS_NAME
    FROM TB_INVENTORY
</select>
```

**예시 3: 페이징 (가장 중요!)**

```xml
<!-- Oracle (HTOMSS) -->
<select id="selectList">
    SELECT * FROM (
        SELECT ROWNUM AS RNUM, A.* FROM (
            SELECT * FROM TB_INVENTORY
            ORDER BY CREATE_DATE DESC
        ) A
        WHERE ROWNUM <= #{pageNo} * #{pageSize}
    )
    WHERE RNUM > (#{pageNo} - 1) * #{pageSize}
</select>

<!-- MariaDB (tomms-lite) -->
<select id="selectList">
    SELECT * FROM TB_INVENTORY
    ORDER BY CREATE_DATE DESC
    LIMIT #{pageSize} OFFSET #{offset}
    <!-- offset은 DTO에서 계산: (pageNo - 1) * pageSize -->
</select>
```

3. ✅ 기본값이 있는 필드는 초기화

### 4-2. Response DTO

**작성 원칙**:
```java
package hsc.tomms.web.im.ir.dto.response;

import java.math.BigDecimal;
import lombok.Getter;
import lombok.Setter;
import lombok.ToString;

@Getter
@Setter
@ToString
public class InventoryListResDto {
    
    /** 재고ID */
    private String invtId;
    
    /** 품목코드 */
    private String itemCode;
    
    /** 품목명 */
    private String itemName;
    
    /** 수량 */
    private int qty;
    
    /** 단가 */
    private BigDecimal unitPrice;
    
    /** 생성일시 */
    private String createDate;
}
```

**포인트**:
1. ✅ 금액 필드는 `BigDecimal` 사용
2. ✅ 날짜 필드는 `String` 사용 (포맷팅 필요)
3. ✅ 필드명은 camelCase (MyBatis alias와 일치)

## ❓ 자주 묻는 질문

### Q1: Record의 모든 필드를 DTO로 만들어야 하나요?

**답변**: 네, HTOMSS의 Record에서 `getString()`, `getInt()` 등으로 추출하던 모든 필드를 DTO 필드로 정의해야 합니다.

### Q2: RecordSet을 항상 List로 변환해야 하나요?

**답변**: 목록 조회는 `List<DTO>`, 단건 조회는 `DTO`, CUD는 `Map<String, Object>` 또는 `int`를 사용합니다.

### Q3: @ServiceId는 어떻게 명명하나요?

**답변**: `{화면코드}/{메서드명}` 형식입니다. 예: `IMIR010/selectInventoryList`

### Q4: Validation은 어디서 하나요?

**답변**: Service 메서드 시작 부분에서 수행하며, 복잡한 경우 private 메서드로 분리합니다.
