## 📝 Step 2: HTOMSS 원본에서 화면 선정

### 2-1. HTOMSS 원본 화면 분석

```bash
cd /htomss/htomss-olt

# 각 모듈의 Service 파일 확인
find htomss-biz-inventory/src -name "*ServiceImpl.java"
find htomss-biz-pbl/src -name "*ServiceImpl.java"
```

### 2-2. 화면 선정 기준

**우선순위**:
1. **단순 조회 화면** (selectList만 있는 화면)
   - CRUD 중 Read만 있어 복잡도 낮음
   - 첫 마이그레이션에 적합

2. **CRUD 화면** (등록/수정/삭제 포함)
   - 기본적인 업무 화면
   - 트랜잭션 처리 필요

3. **복잡한 업무 로직 화면**
   - 여러 테이블 조인
   - 복잡한 비즈니스 로직
   - 외부 연동 등

### 2-3. 선정 화면 예시

**예시: 재고 조회 화면 선정**

**예상 작업**:
- DTO 생성: Request 2개, Response 2개
- Service: interface 1개, Impl 1개
- DAO: interface 1개, Impl 1개
- Mapper XML: 1개
- 예상 기간: 3-4일

## 🔧 디렉토리 생성 스크립트 (선택사항)

선정한 화면의 패키지 구조를 미리 생성할 수 있습니다:

```bash
#!/bin/bash

# 변수 설정
MODULE="im"        # 모듈 (예: im, sp 등)
SCREEN="ir"        # 화면 (예: ir, om 등)
BASE_DIR="/Users/byengju/Documents/01. Project/tommslite/tomms-lite-war/tomms-biz-inventorymgmt"

# Java 디렉토리 생성
JAVA_DIR="$BASE_DIR/src/main/java/hsc/tomms/web/$MODULE/$SCREEN"
mkdir -p "$JAVA_DIR/dto/request"
mkdir -p "$JAVA_DIR/dto/response"
mkdir -p "$JAVA_DIR/service"
mkdir -p "$JAVA_DIR/dao"

# Resources 디렉토리 생성
RESOURCES_DIR="$BASE_DIR/src/main/resources/hsc/tomms/web/$MODULE/$SCREEN/mybatis/mappers"
mkdir -p "$RESOURCES_DIR"

echo "디렉토리 구조 생성 완료!"
echo "Java: $JAVA_DIR"
echo "Resources: $RESOURCES_DIR"
```

## ❓ 문제 해결

### Q1: tomms-lite-war 빌드 실패

**증상**:
```
[ERROR] BUILD FAILURE
[ERROR] Failed to execute goal ... compilation failure
```

**해결 방법**:
1. Java 버전 확인 (Java 11 이상 필요)
2. Maven 버전 확인 (3.6 이상 필요)
3. 의존성 다운로드: `mvn dependency:resolve`
4. Clean 후 재빌드: `mvn clean install`

### Q2: Application 실행 시 포트 충돌

**증상**:
```
Port 8080 is already in use
```

**해결 방법**:
1. 다른 애플리케이션 종료
2. 또는 application.yml에서 포트 변경:
```yaml
server:
  port: 8090
```

### Q3: @ServiceId를 찾을 수 없음

**증상**:
```
cannot find symbol @ServiceId
```

**해결 방법**:
1. tomms-lite-fw 의존성 확인
2. import 구문 확인:
```java
import hone.bom.annotation.ServiceId;
import hone.bom.annotation.ServiceName;
```
