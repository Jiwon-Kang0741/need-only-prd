## ⚠️ 중요: DTO 명명 규칙

**DTO 클래스명은 화면 코드를 붙이지 않습니다**:

예: 화면 코드 `PMDP020` → DTO는 `SearchListReqDto` (화면 코드 없음)

1. **Service/DAO 파일명**: 전체 화면 코드 사용 (`PMDP020Service.java`)
2. **DTO 클래스명**: 화면 코드 없이 기능명만 사용 (`SearchListReqDto.java`)
3. **@ServiceId**: 전체 화면 코드 사용 (`@ServiceId("PMDP020/searchList")`)
4. **패키지 경로**: 화면 코드에 맞는 구조 유지 (`hsc.tomms.web.pm.dp`)
5. **주석/JavaDoc**: 각 파일의 목적에 맞게 작성

### 1-2. Request DTO 작성 예시

**IMIR010 재고 목록 조회 Request**

**파일**: `InventoryListReqDto.java`  
**위치**: `tomms-biz-inventorymgmt/src/main/java/hsc/tomms/web/im/ir/dto/request/`

```java
package hsc.tomms.web.im.ir.dto.request;

import lombok.Getter;
import lombok.Setter;
import lombok.ToString;

/**
 * 재고 목록 조회 요청 DTO
 */
@Getter
@Setter
@ToString
public class InventoryListReqDto {
    
    /** 창고ID (필수) */
    private String storageId;
    
    /** 품목코드 (선택) */
    private String itemCode;
    
    /** 품목명 (선택) */
    private String itemName;
    
    /** 검색어 (선택) */
    private String searchText;
    
    /** 시작일 (선택) */
    private String startDate;
    
    /** 종료일 (선택) */
    private String endDate;
    
    /** 페이지번호 (기본 1) */
    private int pageNo = 1;
    
    /** 페이지크기 (기본 10) */
    private int pageSize = 10;
}
```

### 1-3. Request DTO 작성 원칙

1. **필드 설계**
   - HTOMSS Record의 `getString("key")` → `private String key`
   - HTOMSS Record의 `getInt("key")` → `private int key`
   
2. **필수 필드 표시**
   - 주석으로 `(필수)` 또는 `(선택)` 명시
   
3. **기본값 설정**
   - 페이징: `pageNo = 1`, `pageSize = 10`
   - 필요시 생성자에서 초기화

4. **Validation**
   - 필수 필드는 Service에서 검증
   - `@NotNull`, `@NotEmpty` 등 사용 가능

## 📝 Step 4: DAO Implementation 작성

### 4-1. DAO Impl 작성 예시

**파일**: `IMIR010DaoImpl.java`  
**위치**: `tomms-biz-inventorymgmt/src/main/java/hsc/tomms/web/im/ir/dao/`

```java
package hsc.tomms.web.im.ir.dao;

import hsc.tomms.web.im.ir.dto.request.InventoryListReqDto;
import hsc.tomms.web.im.ir.dto.request.InventorySaveReqDto;
import hsc.tomms.web.im.ir.dto.response.InventoryDetailResDto;
import hsc.tomms.web.im.ir.dto.response.InventoryListResDto;
import lombok.RequiredArgsConstructor;
import org.apache.ibatis.session.SqlSession;
import org.springframework.stereotype.Repository;

import java.util.List;

/**
 * 재고관리 DAO 구현
 */
@Repository
@RequiredArgsConstructor
public class IMIR010DaoImpl implements IMIR010Dao {
    
    private final SqlSession sqlSession;
    
    private static final String NAMESPACE = "hsc.tomms.web.im.ir.dao.IMIR010Dao";
    
    @Override
    public List<InventoryListResDto> selectInventoryList(InventoryListReqDto request) {
        return sqlSession.selectList(NAMESPACE + ".selectInventoryList", request);
    }
    
    @Override
    public InventoryDetailResDto selectInventoryDetail(String invtId) {
        return sqlSession.selectOne(NAMESPACE + ".selectInventoryDetail", invtId);
    }
    
    @Override
    public int insertInventory(InventorySaveReqDto request) {
        return sqlSession.insert(NAMESPACE + ".insertInventory", request);
    }
    
    @Override
    public int updateInventory(InventorySaveReqDto request) {
        return sqlSession.update(NAMESPACE + ".updateInventory", request);
    }
    
    @Override
    public int deleteInventory(String invtId) {
        return sqlSession.delete(NAMESPACE + ".deleteInventory", invtId);
    }
    
    @Override
    public int selectInventoryCount(InventoryListReqDto request) {
        return sqlSession.selectOne(NAMESPACE + ".selectInventoryCount", request);
    }
}
```

### 4-2. DaoImpl 작성 원칙

1. **SqlSession 주입**
   - `@RequiredArgsConstructor` + `private final SqlSession sqlSession`

2. **Namespace 상수**
   - `private static final String NAMESPACE = "패키지.경로.Dao인터페이스명"`

3. **메서드 구현**
   - `selectList`: 목록 조회
   - `selectOne`: 단건 조회
   - `insert`: 등록
   - `update`: 수정
   - `delete`: 삭제

### 5-2. Mapper XML 작성 예시

**파일**: `IMIR010Mapper.xml`  
**위치**: `tomms-biz-inventorymgmt/src/main/resources/hsc/tomms/web/im/ir/mybatis/mappers/`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="hsc.tomms.web.im.ir.dao.IMIR010Dao">
    
    <!-- 재고 목록 조회 -->
    <select id="selectInventoryList" 
            parameterType="hsc.tomms.web.im.ir.dto.request.InventoryListReqDto"
            resultType="hsc.tomms.web.im.ir.dto.response.InventoryListResDto">
        SELECT 
            I.INVT_ID as invtId,
            I.ITEM_CODE as itemCode,
            I.ITEM_NAME as itemName,
            I.STORAGE_ID as storageId,
            S.STORAGE_NAME as storageName,
            I.QTY as qty,
            I.UNIT_PRICE as unitPrice,
            I.QTY * I.UNIT_PRICE as totalAmount,
            I.STATUS_CODE as statusCode,
            C.CODE_NAME as statusName,
            TO_CHAR(I.CREATE_DATE, 'YYYY-MM-DD HH24:MI:SS') as createDate,
            I.CREATE_USER as createUser
        FROM TB_INVENTORY I
        INNER JOIN TB_STORAGE S ON I.STORAGE_ID = S.STORAGE_ID
        LEFT JOIN TB_CODE C ON I.STATUS_CODE = C.CODE AND C.CODE_TYPE = 'INVT_STATUS'
        WHERE 1=1
          AND I.STORAGE_ID = #{storageId}
          <if test="itemCode != null and itemCode != ''">
          AND I.ITEM_CODE = #{itemCode}
          </if>
          <if test="itemName != null and itemName != ''">
          AND I.ITEM_NAME LIKE '%' || #{itemName} || '%'
          </if>
          <if test="searchText != null and searchText != ''">
          AND (I.ITEM_CODE LIKE '%' || #{searchText} || '%' 
               OR I.ITEM_NAME LIKE '%' || #{searchText} || '%')
          </if>
          <if test="startDate != null and startDate != ''">
          AND I.CREATE_DATE >= TO_DATE(#{startDate}, 'YYYY-MM-DD')
          </if>
          <if test="endDate != null and endDate != ''">
          AND I.CREATE_DATE <![CDATA[<=]]> TO_DATE(#{endDate}, 'YYYY-MM-DD') + 1
          </if>
        ORDER BY I.CREATE_DATE DESC
        OFFSET #{pageNo} ROWS FETCH NEXT #{pageSize} ROWS ONLY
    </select>
    
    <!-- 재고 상세 조회 -->
    <select id="selectInventoryDetail" 
            parameterType="string"
            resultType="hsc.tomms.web.im.ir.dto.response.InventoryDetailResDto">
        SELECT 
            I.INVT_ID as invtId,
            I.ITEM_CODE as itemCode,
            I.ITEM_NAME as itemName,
            I.STORAGE_ID as storageId,
            S.STORAGE_NAME as storageName,
            I.QTY as qty,
            I.UNIT_PRICE as unitPrice,
            I.STATUS_CODE as statusCode,
            TO_CHAR(I.CREATE_DATE, 'YYYY-MM-DD HH24:MI:SS') as createDate,
            I.CREATE_USER as createUser,
            TO_CHAR(I.UPDATE_DATE, 'YYYY-MM-DD HH24:MI:SS') as updateDate,
            I.UPDATE_USER as updateUser
        FROM TB_INVENTORY I
        INNER JOIN TB_STORAGE S ON I.STORAGE_ID = S.STORAGE_ID
        WHERE I.INVT_ID = #{invtId}
    </select>
    
    <!-- 재고 등록 -->
    <insert id="insertInventory" 
            parameterType="hsc.tomms.web.im.ir.dto.request.InventorySaveReqDto">
        INSERT INTO TB_INVENTORY (
            INVT_ID,
            ITEM_CODE,
            ITEM_NAME,
            STORAGE_ID,
            QTY,
            UNIT_PRICE,
            STATUS_CODE,
            CREATE_DATE,
            CREATE_USER
        ) VALUES (
            #{invtId},
            #{itemCode},
            #{itemName},
            #{storageId},
            #{qty},
            #{unitPrice},
            #{statusCode},
            SYSDATE,
            #{createUser}
        )
    </insert>
    
    <!-- 재고 수정 -->
    <update id="updateInventory" 
            parameterType="hsc.tomms.web.im.ir.dto.request.InventorySaveReqDto">
        UPDATE TB_INVENTORY
        SET ITEM_NAME = #{itemName},
            QTY = #{qty},
            UNIT_PRICE = #{unitPrice},
            STATUS_CODE = #{statusCode},
            UPDATE_DATE = SYSDATE,
            UPDATE_USER = #{updateUser}
        WHERE INVT_ID = #{invtId}
    </update>
    
    <!-- 재고 삭제 -->
    <delete id="deleteInventory" parameterType="string">
        DELETE FROM TB_INVENTORY
        WHERE INVT_ID = #{invtId}
    </delete>
    
    <!-- 재고 목록 개수 조회 -->
    <select id="selectInventoryCount" 
            parameterType="hsc.tomms.web.im.ir.dto.request.InventoryListReqDto"
            resultType="int">
        SELECT COUNT(*)
        FROM TB_INVENTORY I
        WHERE 1=1
          AND I.STORAGE_ID = #{storageId}
          <if test="itemCode != null and itemCode != ''">
          AND I.ITEM_CODE = #{itemCode}
          </if>
          <if test="itemName != null and itemName != ''">
          AND I.ITEM_NAME LIKE '%' || #{itemName} || '%'
          </if>
          <if test="searchText != null and searchText != ''">
          AND (I.ITEM_CODE LIKE '%' || #{searchText} || '%' 
               OR I.ITEM_NAME LIKE '%' || #{searchText} || '%')
          </if>
    </select>
    
</mapper>
```

### 5-3. Mapper 작성 원칙

1. **Namespace**
   - DAO Interface 전체 패키지 경로 사용

2. **동적 쿼리**
   - `<if test="param != null and param != ''">` 사용
   - 문자열 검색: `LIKE '%' || #{param} || '%'`

3. **날짜 변환**
   - `TO_CHAR(날짜컬럼, 'YYYY-MM-DD HH24:MI:SS')` as 필드명

4. **페이징**
   - Oracle: `OFFSET #{pageNo} ROWS FETCH NEXT #{pageSize} ROWS ONLY`

5. **CDATA**
   - `<`, `>` 등은 `<![CDATA[<=]]>` 로 감싸기

## 📝 Step 7: Service Implementation 작성

### 7-1. Service Impl 작성 예시

**파일**: `IMIR010ServiceImpl.java`  
**위치**: `tomms-biz-inventorymgmt/src/main/java/hsc/tomms/web/im/ir/service/`

```java
package hsc.tomms.web.im.ir.service;

import has.fw.config.annotation.ServiceId;
import has.fw.config.annotation.ServiceName;
import hsc.tomms.web.im.ir.dao.IMIR010Dao;
import hsc.tomms.web.im.ir.dto.request.InventoryListReqDto;
import hsc.tomms.web.im.ir.dto.request.InventorySaveReqDto;
import hsc.tomms.web.im.ir.dto.response.InventoryDetailResDto;
import hsc.tomms.web.im.ir.dto.response.InventoryListResDto;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.HashMap;
import java.util.List;
import java.util.Map;

/**
 * 재고관리 Service 구현
 */
@Slf4j
@Service("IMIR010Service-재고관리")
@RequiredArgsConstructor
public class IMIR010ServiceImpl implements IMIR010Service {
    
    private final IMIR010Dao imir010Dao;
    
    /**
     * 재고 목록 조회
     * API: POST /online/mvcJson/IMIR010/selectInventoryList
     */
    @Override
    @ServiceId("IMIR010/selectInventoryList")
    @ServiceName("재고 목록 조회")
    public List<InventoryListResDto> selectInventoryList(InventoryListReqDto request) {
        log.info("재고 목록 조회 시작: {}", request);
        
        // Validation
        if (request.getStorageId() == null || request.getStorageId().isEmpty()) {
            throw new IllegalArgumentException("창고ID는 필수입니다.");
        }
        
        // 조회
        List<InventoryListResDto> result = imir010Dao.selectInventoryList(request);
        
        log.info("재고 목록 조회 완료: {} 건", result.size());
        return result;
    }
    
    /**
     * 재고 상세 조회
     * API: POST /online/mvcJson/IMIR010/selectInventoryDetail
     */
    @Override
    @ServiceId("IMIR010/selectInventoryDetail")
    @ServiceName("재고 상세 조회")
    public InventoryDetailResDto selectInventoryDetail(String invtId) {
        log.info("재고 상세 조회 시작: {}", invtId);
        
        // Validation
        if (invtId == null || invtId.isEmpty()) {
            throw new IllegalArgumentException("재고ID는 필수입니다.");
        }
        
        // 조회
        InventoryDetailResDto result = imir010Dao.selectInventoryDetail(invtId);
        
        if (result == null) {
            throw new IllegalArgumentException("재고 정보를 찾을 수 없습니다.");
        }
        
        log.info("재고 상세 조회 완료: {}", result);
        return result;
    }
    
    /**
     * 재고 저장 (등록/수정)
     * API: POST /online/mvcJson/IMIR010/saveInventory
     */
    @Override
    @Transactional
    @ServiceId("IMIR010/saveInventory")
    @ServiceName("재고 저장")
    public Map<String, Object> saveInventory(InventorySaveReqDto request) {
        log.info("재고 저장 시작: {}", request);
        
        // Validation
        validateInventorySave(request);
        
        int result = 0;
        if (request.getInvtId() == null || request.getInvtId().isEmpty()) {
            // 등록
            result = imir010Dao.insertInventory(request);
        } else {
            // 수정
            result = imir010Dao.updateInventory(request);
        }
        
        Map<String, Object> response = new HashMap<>();
        response.put("success", result > 0);
        response.put("message", result > 0 ? "저장되었습니다." : "저장 실패");
        
        log.info("재고 저장 완료: {}", response);
        return response;
    }
    
    /**
     * 재고 삭제
     * API: POST /online/mvcJson/IMIR010/deleteInventory
     */
    @Override
    @Transactional
    @ServiceId("IMIR010/deleteInventory")
    @ServiceName("재고 삭제")
    public Map<String, Object> deleteInventory(String invtId) {
        log.info("재고 삭제 시작: {}", invtId);
        
        // Validation
        if (invtId == null || invtId.isEmpty()) {
            throw new IllegalArgumentException("재고ID는 필수입니다.");
        }
        
        // 삭제
        int result = imir010Dao.deleteInventory(invtId);
        
        Map<String, Object> response = new HashMap<>();
        response.put("success", result > 0);
        response.put("message", result > 0 ? "삭제되었습니다." : "삭제 실패");
        
        log.info("재고 삭제 완료: {}", response);
        return response;
    }
    
    /**
     * 재고 저장 Validation
     */
    private void validateInventorySave(InventorySaveReqDto request) {
        if (request.getItemCode() == null || request.getItemCode().isEmpty()) {
            throw new IllegalArgumentException("품목코드는 필수입니다.");
        }
        if (request.getStorageId() == null || request.getStorageId().isEmpty()) {
            throw new IllegalArgumentException("창고ID는 필수입니다.");
        }
        if (request.getQty() < 0) {
            throw new IllegalArgumentException("수량은 0 이상이어야 합니다.");
        }
    }
}
```

### 7-2. Service Impl 작성 원칙

1. **@ServiceId 사용**
   - 각 public 메서드에 `@ServiceId("{화면코드}/{메서드명}")` 어노테이션
   - 자동으로 `/online/mvcJson/{ServiceId}` API 생성

2. **@ServiceName**
   - 메서드 설명을 `@ServiceName` 으로 명시

3. **Validation**
   - Service 레이어에서 비즈니스 검증 수행
   - 필수 파라미터, 범위 체크 등

4. **@Transactional**
   - CUD 작업은 `@Transactional` 적용

5. **로깅**
   - `@Slf4j` + `log.info()` 사용
   - 시작/완료 로그 출력

6. **응답 형식**
   - CUD: `Map<String, Object>` 로 success/message 반환
   - 조회: DTO 또는 List<DTO> 반환

## ❓ 문제 해결

### Q1: @ServiceId가 인식되지 않아요

**답변**:
- Framework 모듈에 @ServiceId 어노테이션이 정의되어 있는지 확인
- tomms-fw-base 의존성이 추가되어 있는지 확인

### Q2: MyBatis Mapper를 찾을 수 없어요

**답변**:
- Mapper XML 위치가 `src/main/resources/hsc/tomms/web/{module}/{screen}/mybatis/mappers/` 인지 확인
- namespace가 DAO Interface 전체 경로와 일치하는지 확인

### Q3: API 호출이 404 에러가 나요

**답변**:
- @ServiceId 값 확인
- API 경로: `/online/mvcJson/{ServiceId}`
- 예: `/online/mvcJson/IMIR010/selectInventoryList`

### Q4: 트랜잭션이 동작하지 않아요

**답변**:
- Service Impl에 `@Transactional` 어노테이션 추가
- Boot Application에 `@EnableTransactionManagement` 확인
